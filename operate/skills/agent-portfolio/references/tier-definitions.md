# Tier 정의 상세

## T1 — 사업 핵심
- **정의**: 중단 시 매출·고객 신뢰가 즉시 손상
- **운영**: 24/7 모니터, 인시던트 즉시 대응, P0/P1 알람
- **갱신 주기**: 변경 시 페어 리뷰 + 단계 배포 필수
- **예시**: 결제 라우터, 고객 응대 1차, 인시던트 라우터
- **권장 비율**: ≤ 20%

## T2 — 사업 중요
- **정의**: 매일 실행되며 내부 의사결정 의존
- **운영**: 매일 헬스체크, 4~24시간 SLA
- **예시**: 데일리 브리핑, 비용 모니터, 주간 우선순위
- **권장 비율**: 30~40%

## T3 — 운영 효율
- **정의**: 주기적 자동화, 사람 검토 후 사용
- **운영**: 주 1회 리뷰
- **예시**: 콘텐츠 큐레이션, 주간 회고, PRD 어시스트
- **권장 비율**: 30~40%

## T4 — 실험
- **정의**: 신규 또는 검증 단계, 운영 자산이 아님
- **운영**: 격주 리뷰 + 자체 평가
- **승격 기준**: 4주 연속 성공 + 사용자 ≥ N명 → T3 승격
- **권장 비율**: ≤ 10%

## T5 — sunset 후보
- **정의**: 사용 빈도 < 월 1회 또는 대체 가능
- **운영**: 월 1회 점검 → 삭제 결정
- **권장 비율**: ≤ 5% (있으면 빠르게 처리)

## 가중치 (인시던트 우선순위 계산용)

| 티어 | 가중치 |
|---|---|
| T1 | 5.0 |
| T2 | 3.0 |
| T3 | 1.5 |
| T4 | 0.8 |
| T5 | 0.2 |

우선순위 점수 = (이번 주 인시던트 수) × 티어 가중치

→ 점수 내림차순으로 운영 주의력 분배
