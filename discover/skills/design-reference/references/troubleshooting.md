# design-reference — 자주 발생하는 문제

## 문제 1: 타겟 카테고리가 리스트에 없다
상황: 사용자 타겟이 "법률 서비스"처럼 site-list에 없는 카테고리일 때.
해결:
1. 가장 유사한 카테고리로 필터 (법률 → 헬스케어 또는 생산성·SaaS)
2. 사용자에게 대체 카테고리 확인
3. 글로벌 리스트에서 해당 도메인 사이트를 WebSearch로 보완 (SKILL.md 허용 도구 확인 후)

## 문제 2: 5-7개 선택이 한 카테고리에 몰린다
상황: 핀테크 타겟인데 핀테크 사이트만 5개 선택됨.
해결:
- 타겟 사용자가 자주 쓰는 다른 앱 1-2개를 반드시 포함 권고
- 예: 핀테크 → 소셜·커뮤니티 or 생산성·SaaS 1개 추가
이유: 사용자의 디자인 기대값은 그들이 주로 쓰는 모든 앱에서 형성됨.

## 문제 3: DESIGN_BRIEF.md가 너무 추상적으로 생성됨
상황: "모던하고 심플하게"같은 근거 없는 결론이 나올 때.
해결:
- 컬러는 반드시 헥스코드 + WCAG 대비비 포함
- 타이포그래피는 반드시 스케일 비율과 기본 크기(px) 포함
- "느낌" 표현은 반드시 구체적 수치나 레퍼런스 사이트로 뒷받침
기준: DESIGN_BRIEF.md 검증 — 이 문서만 보고 디자이너가 토큰 값을 정할 수 있는가?
