# Good Example — build-or-buy

## 사용자 요청
"문서 요약 에이전트를 만들려는데, 직접 개발할지 Glean 같은 기존 솔루션을 쓸지 고민이야. 우리 팀은 엔지니어 2명이고 3개월 안에 런칭해야 해."

## 승인 이유
- 에이전트 단위로 Build/Buy 선택 필요한 상황
- 실제 외부 솔루션(Glean) 존재 확인 가능
- 시간 제약(3개월), 리소스 제약(엔지니어 2명) 명확
- 의사결정을 돕기 위한 데이터 기반 평가 필요

## 예상 처리
1. 요구사항 명확화 (문서 요약 에이전트의 핵심 기능 정의)
2. 차별화 판단 (문서 요약이 우리 제품의 핵심 차별화인가?)
3. 외부 솔루션 조사 (Glean 포함 2~3개 솔루션 비교)
4. 평가 매트릭스 적용 (6개 항목 × 3개 선택지)
5. 가중치 적용 (속도가 우선이므로 2배 가중치)
6. 최종 권장 + 근거 3가지
7. 재검토 트리거 정의 (6개월 후 또는 유지보수 시간 > 20시간)
