# Good Example — PM-ENGINE-MEMORY 동적 검색

## 사용자 요청
"우리가 새로운 에이전트를 만들지, 기존 도구를 사용할지 결정해야 하는데, PM-ENGINE-MEMORY에서 관련 TK를 찾아서 이 상황에 어떻게 적용하는지 보여줘."

## 승인 이유
- pm-engine의 핵심 기능: TK를 동적으로 검색하여 현재 의사결정에 즉시 참조
- 제품 개발 초기 단계에서 Build vs Buy 판단이 필요한 상황
- PM-ENGINE-MEMORY의 TK-007 (Build vs Buy 2주 법칙), TK-003 (에이전트 비용 10배 법칙) 등이 직접 관련

## 예상 처리
1. 사용자의 상황을 분석: "3주 예상 개발 시간, MVP 기능 정의 완료, 기존 오픈소스 도구 3개 존재"
2. PM-ENGINE-MEMORY에서 "build-or-buy" 관련 TK 동적 검색 → TK-007, TK-003 로드
3. TK-007의 "2주 법칙" 적용: 3주 > 2주 → "Buy 우선 검토" 결론
4. TK-003 연관 조회: "비용 시뮬레이션" → 구매가 기존 도구 $2K/년 vs 개발 $15K 예상
5. 최종 판단 근거 제시: "TK 기반 의사결정 → Buy 우선 검토하되, 핵심 차별화 기능이면 Build도 병행 검토"
6. Instruction 업데이트 제안: 다음 신규 기능부터 이 패턴 자동 적용
