# PM-ENGINE-MEMORY Starter Kit

> 이 파일은 pm-engine 스킬의 **빠른 시작 템플릿**입니다.
> 아래 5개의 Seed TK를 복사한 뒤, 자신의 경험으로 수정/추가하세요.
> 실제 사용 시 이 파일을 `PM-ENGINE-MEMORY.md`로 저장합니다.

---

## 사용법

1. 이 파일을 프로젝트 루트에 `PM-ENGINE-MEMORY.md`로 복사
2. Seed TK의 `[내용]` 부분을 자신의 경험으로 교체
3. 매일 1개씩 TK를 추가 — `/pm-tacit-extract` 명령 활용
4. pm-engine 스킬이 자동으로 이 파일을 참조합니다

---

## Seed TK (5개)

### TK-001: 긴급 요청 우선순위 판단

- **핵심 원칙**: 긴급(Urgent)과 중요(Important)를 분리한다. "CEO가 말했으니까"는 긴급이지 중요가 아닐 수 있다.
- **판단 기준**: ①사용자 영향 범위 ②매출 직결 여부 ③되돌릴 수 없는 정도(Irreversibility)
- **🟢 활성화 조건**: 이해관계자가 "급하다"고 말할 때
- **🔗 연관 TK**: TK-004
- **출처**: [프로덕트 매니저로서 3년간 200건+ 긴급 요청 처리 경험]

### TK-002: AI 네이티브 사고 필터

- **핵심 원칙**: 새 기능을 기획할 때 "AI가 이 작업의 80%를 할 수 있는가?"를 첫 번째로 묻는다.
- **판단 기준**: ①반복성 ②패턴 존재 여부 ③오류 허용 범위
- **🟢 활성화 조건**: 새 기능 또는 워크플로우를 설계할 때
- **🔗 연관 TK**: TK-003, TK-005
- **출처**: [AI SaaS 제품 3개 런칭 경험에서 도출]

### TK-003: 에이전트 비용 10배 법칙

- **핵심 원칙**: AI 에이전트의 운영 비용은 초기 예상의 10배가 된다. API 비용뿐 아니라 모니터링, 오류 복구, 엣지 케이스 핸들링을 반드시 포함한다.
- **판단 기준**: ①API 토큰 비용 ②재시도/재처리 비율 ③인간 개입 빈도
- **🟢 활성화 조건**: 에이전트 ROI를 계산하거나 비용 시뮬레이션할 때
- **🔗 연관 TK**: TK-002
- **출처**: [AI Dubbing 서비스 운영 1년차 비용 분석]

### TK-004: "2주 법칙" — MVP 기능 스코핑

- **핵심 원칙**: MVP 기능 하나가 2주를 넘으면 스코프가 잘못 잡힌 것이다. 2주 안에 끝나는 단위로 쪼개라.
- **판단 기준**: ①기능 단위가 독립적으로 릴리즈 가능한가 ②A/B 테스트 가능한가 ③유저가 가치를 느끼는가
- **🟢 활성화 조건**: 스프린트 플래닝 또는 기능 스코핑 시
- **🔗 연관 TK**: TK-001
- **출처**: [Agile PM 5년, 120+ 스프린트 운영]

### TK-005: 데이터 없는 가설은 가설이 아니다

- **핵심 원칙**: "유저가 원할 것 같다"는 가설이 아니라 희망이다. 최소한 정성 데이터(인터뷰 3건) 또는 정량 데이터(이벤트 로그 1주)가 있어야 가설이다.
- **판단 기준**: ①근거 유형(정성/정량/없음) ②샘플 수 ③반복 확인 가능성
- **🟢 활성화 조건**: PRD 작성 시 "문제 정의" 단계
- **🔗 연관 TK**: TK-002
- **출처**: [Data-Driven PM 방법론 + 유저 리서치 50건+ 경험]
