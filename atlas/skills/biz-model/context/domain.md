# 도메인 컨텍스트 — Business Model

## 1) Domain Scope
AI 에이전트 제품의 **가치 창출(value creation)과 가치 포획(value capture) 설계** — 수익 모델, 가격 책정, 비용 구조, 단위 경제.
- 포함: 가치 창출 분석, 비용 구조, 가격 모델 선택, 이윤율, 고객 검증
- 제외: GTM 전략 (forge/oracle 스킬), 시장 규모 분석 (oracle 스킬), 운영 메트릭 추적 (argus 스킬)

## 2) Primary Users
- **CFO/전략 책임자**: 비즈니스 모델 검증, 수익성 계획
- **PM**: 고객 가치 기반 가격 책정 결정
- **사업가/창업가**: VC 피칭 및 비즈니스 케이스 수립

## 3) Required Inputs
- 에이전트 제품이 고객에게 제공하는 가치:
  - 시간 절감 (시간 × 시급)
  - 에러 감소 (오류 비율 × 비용)
  - 새로운 능력 (정량화 가능 형태로)
- 변수 비용 구조:
  - LLM API 가격
  - 외부 API 호출
  - 인프라 비용
  - 인력 지원 비용
- 고객 사용 패턴:
  - 월간 요청 수
  - 고객 세그먼트별 차이 (스타트업 vs 대기업)
- 경쟁 환경:
  - 경쟁 제품 가격
  - 대체 솔루션 (수동, 아웃소싱 등) 비용

## 4) Output Contract
- **가치 창출 분석**: 3가지 이상 가치 요소 정량화
- **비용 구조**: CPE (Cost Per Execution) 계산 및 각 요소 명시
- **가격 모델**: 선택한 모델 (Per-execution / Subscription / Outcome-based 등) + 근거
- **이윤율**: Gross Margin 계산 (목표: >70%)
- **고객 검증**: 최소 2명 이상 고객 또는 예상 고객과의 타당성 확인
- **Business Model Canvas**: 가치, 비용, 수익을 시각화한 요약

## 5) Guardrails
- 고객 가치 계산 없이 가격 책정 금지 (추측 기반 모델 위험)
- CPE 계산 없이 가격 모델 선택 금지 (수익성 검증 불가)
- Gross Margin <50%는 지속 불가능 (초기 단계도 50% 최소)
- 단일 가격 모델만 검토하고 선택 금지 (최소 3가지 비교)
- 고객 검증 없이 모델 확정 금지 (시장 현실과 괴리 위험)

## 6) Working Facts (TO BE UPDATED by reviewer)
- [ ] 전형적인 SaaS Gross Margin: >70% (AI 에이전트는 변수 비용 때문에 더 낮을 수 있음: 50-65%)
- [ ] CPE 계산 정확도: 초기 예측 대비 실제 사용량 편차 20-40% (모니터링 필수)
- [ ] 고객이 느끼는 가치 vs 우리가 청구하는 가격: 보통 10-20% 비율 (너무 높으면 거부, 너무 낮으면 기회 손실)
- [ ] 가격 모델별 전환율: Per-execution (5-10%), Subscription (20-30%), Outcome-based (40-50% but 속성 어려움)
- [ ] 초기 고객 수집 시간: 보통 2-4주, 가격 타당성 확인 피드백: 1-2주
