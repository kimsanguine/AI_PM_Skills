# Troubleshooting — cohort 스킬

## 1) 코호트 샘플 크기 부족

**증상:**
- "주간 데이터가 100건 미만이어서 신뢰도가 낮다"
- 주간 코호트 분석 결과가 일관되지 않음
- 통계 신뢰도 부족 경고 반복

**확인:**
- 코호트당 주간 표본 수 계산 (최소 100건 권장)
- 각 코호트의 데이터 수집 기간 확인
- 차분 탐지에 필요한 최소 표본 크기 계산

**조치:**
1. 추적 기간 연장:
   - 주간 → 월간 코호트로 변경 (표본 크기 4배 증가)
   - 또는 2주 코호트로 조정
2. 코호트 정의 재검토:
   - 기존: 일일/주간 단위
   - 변경: 월간 또는 배치 단위
3. 신뢰도 명시:
   - "표본 크기: N건, 표준오차: ±X%"로 불확실성 전달
4. 의사결정 기준 수정:
   - 표본 부족 상태에서는 명확한 추세 변화(2-3주 연속)까지만 대응

## 2) 급락 탐지 지연

**증상:**
- "v1.1 정확도가 -5% 떨어졌는데 4주 후에 발견했다"
- 초기에 작은 신호를 놓쳤다가 나중에 큰 문제로 드러남
- "더 빨리 감지할 방법이 없나?"

**확인:**
- 현재 모니터링 빈도 (주간, 월간, 분기?)
- 자동 알림 설정 여부 (Accuracy -2%/주 임계값)
- 초기 신호를 캐치한 주간 리뷰 미팅 여부

**조치:**
1. 자동 모니터링 설정:
   - 일일 또는 주간 자동 계산 및 대시보드 업데이트
   - Accuracy -2%/주 초과 시 자동 알림
2. 주간 리뷰 미팅 설정 (고정 일정, 15분):
   - 모든 코호트의 주간 추이 확인
   - 임계값 이탈 여부 판단
3. 급락 감지 후 immediate action:
   - 원인 분석 시작 (incident 스킬로 전환)
   - 필요시 즉시 rollback
4. 경보 단계 설정:
   - Yellow: -1% ~ -2%/주 (관찰 중)
   - Red: < -2%/주 (즉시 조사)

## 3) 외부 변수와 에이전트 변화 혼동

**증상:**
- "v2.0 성능이 내려갔는데 입력 데이터 난이도가 올라갔을 수도 있다"
- 성능 변화의 원인이 불명확
- "에이전트를 바꿨나, 데이터가 바뀌었나?"

**확인:**
- 성능 저하 기간의 입력 데이터 분포 분석 (난이도, 카테고리, 언어)
- v1.1과 v2.0이 같은 입력으로 테스트되었나?
- 외부 의존성 변화 (외부 API, 데이터 소스)

**조치:**
1. 입력 데이터 분석:
   - 성능 저하 기간의 데이터 특성 추출
   - "이전 기간과 입력이 정말 다른가?" 검증
2. 동일 입력 비교:
   - 동일 입력 데이터셋으로 v1.1 vs v2.0 재테스트
   - A/B 테스트로 버전 간 차이 정량화
3. 데이터 드리프트 모니터링:
   - 주간 입력 데이터 분포 변화 추적
   - 이상 탐지 (분포 변화 임계값)
4. 결론 도출:
   - "에이전트 탓: X%, 데이터 탓: Y%" 분리 평가

## 4) 리텐션 해석 오류

**증상:**
- "Week 1 리텐션 30%를 '저조'라고 판단했는데 다른 버전도 같은 수준이더라"
- 절대값 해석으로 잘못된 의사결정
- "이게 좋은 건가 나쁜 건가 정말 모르겠다"

**확인:**
- 리텐션 해석의 비교 대상 (이전 버전, 경쟁사, 업계 벤치마크)
- 에이전트 특성상 예상되는 리텐션 범위
- Week 1 vs Week 4 리텐션 차이 (급락 vs 점진적)

**조치:**
1. 비교값 수집:
   - 이전 버전: v1.0 Week 1 리텐션 25% 기록 확인
   - 경쟁사 또는 업계 벤치마크 조사
   - 수동 프로세스와 비교 (자동화로 30% 시간 절감 = 리텐션 목표)
2. 리텐션 곡선 분석:
   - Week 1: 30%
   - Week 4: 15% (급락 = 초기 품질 문제)
   - Week 12: 8% (점진적 = 정상 사용 패턴)
3. 해석 기준 수립:
   - < 20% Week 1: 온보딩 또는 초기 품질 문제
   - 20-40% Week 1: 정상 범위
   - > 40% Week 1: 매우 높음 (주의: 과도한 사용)
4. 의사결정:
   - "v2.0 리텐션 30%는 v1.0 대비 +20% 개선" 명시

## 5) 계절성 패턴 미인식

**증상:**
- "월초는 사용량이 많으니까 정확도도 높을 거다" (잘못된 가정)
- 월말 급락을 "정확도 악화"로 착각
- 보통 변동을 문제로 과하게 반응

**확인:**
- 지난 3개월 데이터에서 주기적 패턴 확인 (히스토그램)
- 월초/중순/말 정확도 추이 (평균값)
- 요일별 패턴 (주중 vs 주말)

**조치:**
1. 패턴 학습:
   - 3개월 이상 데이터로 월별/주별 정상 범위 계산
   - Moving average (4주 이동 평균) 추가 계산
2. 정상 범위(밴드) 설정:
   - 월초 정상: 94% ± 2%
   - 월중 정상: 92% ± 2%
   - 월말 정상: 90% ± 3% (사용자 편차 증가)
3. 알림 기준 수정:
   - 범위 내: 알림 없음
   - 범위 이탈 > 3%: "비정상 신호" 알림
4. 의사결정 수정:
   - "월말 90% 정확도" = 정상 (계절성 고려)
   - "월중 88% 정확도" = 비정상 (범위 이탈)
