# Good Example — Burn Rate

## 사용자 요청

"에이전트 운영 비용이 예산을 초과하고 있어. 모델별, 태스크별, 사용자 세그먼트별로 토큰 비용을 분석하고 최적화 방안을 제시해줄 수 있어? 현재는 Sonnet을 쓰는데 Haiku로 다운그레이드할 때 정확도 영향도 평가해줘."

## 승인 이유

- 비용 기준선 측정 (모델별/태스크별/세그먼트별)
- 최적화 전략 검토 (Haiku 다운그레이드 + 트레이드오프)
- 정량화된 분석 요청

## 예상 처리

1. 현재 비용 기준선 계산 (입출력 토큰 · 모델 · 가격)
2. 비용 구성 요소 분해 (시스템 프롬프트, 메모리, 출력)
3. 최적화 전략 평가 (Prompt compression, Response caching, Model downgrade 등)
4. Model downgrade의 정확도 영향 평가 (A/B 테스트)
5. 월간 예산 설정 및 alert 임계값
6. Cost-benefit 분석 및 의사결정
