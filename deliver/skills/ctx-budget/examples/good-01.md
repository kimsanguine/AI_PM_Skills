# Good Example — ctx-budget 스킬

## 사용자 요청

"pm-briefing 에이전트가 여러 파일(SOUL.md, MEMORY.md, 메모리 파일 3개)을 로드하는데, 토큰이 낭비되는 것 같아. 어떤 파일을 항상 로드하고, 어떤 파일을 조건부로 로드해야 할지 계획해줄래?"

## 승인 이유

- 여러 파일 로드로 인한 토큰 낭비 및 응답 품질 저하 우려
- 필수/조건부/제외 분류 필요
- 동적 로딩 전략으로 컨텍스트 최적화

## 예상 처리

1. 로드 파일 목록 작성: SOUL.md, MEMORY.md, 일별 메모리 × 3, SKILL.md × 2 등
2. 토큰 추정: 각 파일별 용량 × 토큰 환산율, 총합 vs 모델 한도 비교
3. 필수/조건부/제외 분류: 필수 4,500 tokens / 조건부 2,400 tokens (기본) / 제외 항목
4. 동적 로딩 전략: memory_search로 관련 파일만 로드하는 조건 정의
5. 모니터링 계획: 70%/85% 임계값별 알림 설정

## 최종 결과물

컨텍스트 윈도우 예산 계획으로 토큰 낭비를 줄이고 응답 품질 유지
