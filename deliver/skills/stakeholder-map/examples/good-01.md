# Good Example — stakeholder-map 스킬

## 사용자 요청

"비용 분석 에이전트를 조직 전체에 도입하려는데, 이해관계자별 저항을 어떻게 극복할지 전략을 짜줄래? 경영진, 사용자, 엔지니어링, 법무, 회계팀, CTO 모두 관련돼있어."

## 승인 이유

- 에이전트 도입이 조직 전체에 영향 미침
- 내부 저항과 블로킹 가능성 예상
- 여러 팀의 합의가 필요

## 예상 처리

1. Stakeholder Identification: 6개 그룹 식별 및 관심사 맵핑
2. Power-Interest Matrix: 각 이해관계자를 4사분면에 배치
3. Resistance Analysis: 저항 유형 3개(Trust Deficit, Control Loss, Risk Aversion) 분석
4. Champion Strategy: PM 담당자를 champion으로 발굴, 무장(데이터/데모/ROI)
5. Communication Plan: 이해관계자별 메시지/포맷/주기 정의
6. Go/No-Go Confidence: 도입 가능성 평가

## 최종 결과물

이해관계자별 맞춤 전략으로 에이전트 도입을 조직적으로 추진 가능
