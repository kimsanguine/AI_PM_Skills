# Bad Example — stakeholder-map 스킬

## 사용자 요청

"이해관계자 관리 전략 세워줄래?"

## 거절 이유

- 어떤 에이전트 도입인지 불명확
- 관련된 이해관계자가 명확하지 않음
- 조직 구조/문화 정보 부족

## 올바른 라우팅

1. 에이전트 이름과 도입 목표 확인
2. 관련된 팀/부서 목록 제공
3. 조직 크기/문화 정보 (필요시)
4. 스킬 호출

## 수정 방향

"비용 분석 에이전트(cost-analyst)를 회사 전체에 도입하려고 해. 이해관계자 맵핑과 저항 극복 전략을 짜줄래? 관련팀: 경영진(CFO), 직접 사용자(PM 담당자), 엔지니어링, 법무, 회계, CTO. 특히 회계팀의 데이터 보안 우려와 법무팀의 AI 규제 우려가 큼."
