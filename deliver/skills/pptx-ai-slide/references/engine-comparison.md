# PPTX 엔진 4종 비교표 (v0.7.0+)

> hplan의 단일 PPTX 흐름은 청중·규모·스타일에 따라 부적합한 경우가 많다.
> 실전 운영에서 검증된 4개 엔진의 트레이드오프와 라우팅 신호를 정리한다.

---

## 1. mckinsey 엔진

- **베이스**: PptxGenJS / Node.js
- **타깃**: Navy + Steel Blue 팔레트, Georgia + Malgun Gothic 타이포
- **적합**:
  - 30장 이상 강의/리포트 시리즈
  - 동일 디자인 시스템으로 반복 생산
  - 컨설팅 톤(임원 보고 등)
- **부적합**:
  - 슬라이드별 커스텀 비주얼
  - 미니멀/에디토리얼 스타일

### 라우팅 신호
- `"30장", "40장", "강의자료", "주차별", "리포트 시리즈"`
- `"맥킨지", "컨설팅 스타일", "임원 보고"`

### Boundary
- 슬라이드 1장만 손보고 싶으면 사용 금지 → html-qa로 이관
- 영상 입력은 video 엔진의 후속 체인으로 사용

---

## 2. hifidelity 엔진

- **베이스**: 슬라이드 디자이너 4-agent 파이프라인 + Gemini Imagen 4.0
- **적합**:
  - 1~10장 투자 피치
  - 연구 발표 (개념 일러스트 자동 생성)
  - 슬라이드별 히어로 이미지 필요
- **부적합**:
  - 30장 이상 대량 생산
  - Imagen 비용이 부담스러운 경우
  - 텍스트 위주 슬라이드

### 라우팅 신호
- `"--with-images", "이미지 같이", "콘셉트 다이어그램 이미지"`
- `"피치덱", "투자자 미팅", "연구 발표 8장"`

### Boundary
- 텍스트만 필요하면 html-qa로 이관
- 대량(20+)이면 mckinsey + 선택 슬라이드만 hifidelity 하이브리드 고려

---

## 3. html-qa 엔진 (디폴트)

- **베이스**: HTML-first → Playwright PDF → 100점 루브릭 QA
- **테마**: mckinsey / minimal / editorial (CSS 스왑)
- **적합**:
  - 5~25장 일반 케이스 (디폴트)
  - 슬라이드별 디자인 자유도 필요
  - 자동 QA 게이트 필요
  - 기존 PPTX 리디자인(no-delete 제약)
- **부적합**:
  - 30장 이상 대량 (mckinsey가 더 빠름)
  - 히어로 이미지 자동 생성 (hifidelity가 더 적합)

### 라우팅 신호
- 명시적 신호 없으면 **이 엔진을 선택**한다 (디폴트)
- `"미니멀", "에디토리얼", "리디자인", "QA", "100점"`

### Boundary
- 영상 입력은 직접 받지 않음 → video → html-qa 체인

---

## 4. video 엔진 (전처리)

- **베이스**: yt-dlp / 로컬 mp4 → Whisper 전사 → 섹션화 → 슬라이드 brief
- **적합**:
  - YouTube URL, 로컬 mp4를 슬라이드로 전환
  - 강의 녹화를 교안으로 재가공
- **부적합**:
  - 텍스트 직접 입력 (다른 엔진 직접 호출)
- **체이닝 필수**:
  - 강의 시리즈면 → mckinsey
  - 일반 요약이면 → html-qa
  - 비주얼 강조면 → hifidelity

### 라우팅 신호
- 입력에 URL 패턴(`youtu.be`, `youtube.com`) 또는 `.mp4`, `.m4a`, `.wav`
- `"녹화", "전사", "강의 영상", "유튜브"`

### Boundary
- 단독으로는 PPTX를 생산하지 않는다. 항상 후속 엔진이 필요하다.

---

## 결정 트리 (한 장)

```
START
 │
 ├── 영상 입력? ──── YES ──→ video (필수) → 후속 엔진 결정으로 복귀
 │     └── NO
 │
 ├── 장수 ≥ 30 또는 강의 시리즈? ──── YES ──→ mckinsey
 │     └── NO
 │
 ├── 장수 ≤ 10 AND (피치/연구/이미지 필수)? ──── YES ──→ hifidelity
 │     └── NO
 │
 └── 디폴트: html-qa (5~25장, 미니멀/에디토리얼/QA)
```

## 운영 노하우

1. **엔진 미설치 환경**: hplan 단독으로는 4엔진을 실행하지 않는다.
   라우터가 엔진 이름과 입력 명세까지만 제시하고, 실제 생성은 사용자의
   설치된 도구 체인(`pptx-mckinsey`, `pptx-hifidelity`, `pptx-html-qa`, `pptx-video`
   subagent)이 수행한다.
2. **하이브리드 케이스**: 30장 강의 중 핵심 5장만 hifidelity로 이미지 강화 가능.
   → mckinsey + 선택 슬라이드 hifidelity overlay
3. **비용 게이트**: hifidelity는 Imagen 호출 비용이 크므로 hplan `cogs-sentinel`로
   사전 검증 권장.
