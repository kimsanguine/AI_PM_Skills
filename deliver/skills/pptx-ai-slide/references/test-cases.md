# Test Cases — PPTX AI Slide

## 1) Trigger Tests

### Should Trigger (5)

1. "고객 문의 분류 에이전트를 C-Level에 피칭할 발표자료 만들어줄 수 있을까? 핵심은 비용 절감과 응답 속도 개선이야. 7장 이내로."
   - 이유: 청중(C-Level) + 주제 + 핵심 메시지 + 분량이 명확

2. "아키텍처 다이어그램을 슬라이드로 설명해야 하는데, 엔지니어링 팀 대상이야. 시스템 흐름이 명확하게 보이도록 만들어줄 수 있을까?"
   - 이유: 기술 슬라이드 구성 필요

3. "우리 AI 에이전트 프로젝트의 피치 덱을 만들어줄 수 있을까? 투자자 대상이고, 차별화 포인트와 시장 기회를 강조해야 해."
   - 이유: 투자자 피칭용 스트럭처 필요

4. "스타트업 CEO를 대상으로 에이전트 데모 발표를 준비 중이야. 슬라이드 8-10장으로, 문제 → 솔루션 → 데모 순서로."
   - 이유: 청중별 슬라이드 구조 설계 필요

5. "분기별 이해관계자 업데이트 발표를 준비해야 하는데, 비기술 청중이야. 현재 진행률, ROI, 다음 단계를 담아서 만들어줄 수 있을까?"
   - 이유: 이해관계자 커뮤니케이션 슬라이드 필요

### Should NOT Trigger (5)

1. "에이전트 아키텍처를 설계해주세요."
   - 올바른 라우팅: `orchestration` 또는 `3-tier`

2. "발표 스크립트(스크립트만) 작성해주세요."
   - 올바른 라우팅: 글쓰기 도구 또는 일반 프롬프트 (이 스킬은 PPTX 결과물에 집중)

3. "에이전트 설계 변경 시 영향도를 분석해주세요."
   - 올바른 라우팅: `agent-plan-review`

4. "이 슬라이드를 영상으로 변환해주세요."
   - 올바른 라우팅: `agent-demo-video`

5. "이해관계자 커뮤니케이션 전략을 세워주세요."
   - 올바른 라우팅: `stakeholder-map`

## 2) Functional Tests (Given-When-Then)

### Test 1: C-Level 대상 피치 덱 (1메시지/장, 7장 이내)

**Given:**
- 에이전트: expense-analyzer (비용 분석 자동화)
- 청중: CFO + CMO (C-Level 2명)
- 핵심 메시지: "월 $500 비용 절감 + 의사결정 속도 50% 향상"
- 분량: 5-7장

**When:**
- `/pptx-ai-slide expense-analyzer --audience C-Level --slides 7` 스킬 실행

**Then:**
- 슬라이드 구조:
  1. 문제: "매달 비용 분석에 몇 시간?" (구체적 수치)
  2. 솔루션: "에이전트로 자동화"
  3. 아키텍처: 시스템 흐름 다이어그램 (1개)
  4. 데모/결과: 실제 동작 또는 비교 표
  5. 비용/ROI: "$500/월 절감 + ROI 차트"
  6. 리스크 & 완화: "데이터 보안 대책 ✅"
  7. 로드맵: "3개월 내 배포"
- 각 슬라이드에 1개 메시지만 있음 (bullet 최대 5개)
- 텍스트 overflow 0건
- C-Level 대상이므로 기술 깊이는 낮음 (ROI/시간/정성적 가치 중심)

---

### Test 2: 엔지니어링 팀 대상 기술 슬라이드 (높은 메시지 밀도, 다이어그램 다수)

**Given:**
- 에이전트: news-aggregator (뉴스 수집 + 요약)
- 청중: 엔지니어링 팀 (10-15명)
- 핵심: 아키텍처 + 기술 스택 + 배포 방식
- 분량: 12-15장

**When:**
- 기술 슬라이드 구성 요청

**Then:**
- 슬라이드 구조:
  1. 시스템 아키텍처: Mermaid flowchart (에이전트 오케스트레이션)
  2. 데이터 흐름: Mermaid sequenceDiagram
  3. 기술 스택: "Python 3.10, Claude API, PostgreSQL"
  4. 모듈별 상세: 각 모듈의 책임 + 인터페이스
  5. 에러 처리: FMEA 매트릭스
  6. 배포 파이프라인: CI/CD 다이어그램
  7. 모니터링: KPI 수집 지점
- 다이어그램 4개 이상 포함 (text-only 슬라이드 최소화)
- 메시지 밀도 높음 (슬라이드당 3-4가지 정보)

---

### Test 3: 슬라이드 분할 필요성 감지 (1메시지 원칙 위배 시 분할 제안)

**Given:**
- 제시된 슬라이드: "아키텍처 + 데이터 흐름 + 에러 처리를 1장에 설명"
- 문제: 텍스트/다이어그램 overflow + 메시지가 3개

**When:**
- 1메시지/장 원칙 점검

**Then:**
- 분할 제안:
  ```
  슬라이드 1: 아키텍처 (flowchart only)
  슬라이드 2: 데이터 흐름 (sequenceDiagram only)
  슬라이드 3: 에러 처리 (FMEA 테이블 or 상세 설명)
  ```
- "1슬라이드 1메시지 원칙을 따르면 정보 흡수율이 80% 향상됩니다" 설명

## 3) Error Cases

### Error 1: 청중/목적 불명확 → 슬라이드 구조 결정 불가

**상황:**
- 요청: "발표자료 만들어줄 수 있을까?"
- 청중/목적/분량/핵심 메시지 모두 불명확

**대응:**
1. "다음 4가지를 먼저 결정해주세요:"
   - 청중: C-Level / PM팀 / 엔지니어링 / 투자자 중?
   - 목적: 이해도? 승인? 피칭? 데모?
   - 분량: 몇 장?
   - 핵심 메시지: 3개 이내로?
2. 선택지 각각에 대해 권장 슬라이드 수 제시

---

### Error 2: 아키텍처 슬라이드에 다이어그램 없이 텍스트만 존재

**상황:**
- "아키텍처: 사용자 → API → 에이전트 → 데이터베이스 → 응답"을 텍스트로만 설명

**대응:**
1. "아키텍처는 반드시 시각화해야 합니다"로 명시
2. Mermaid 다이어그램 제안:
   ```mermaid
   flowchart LR
       User["사용자"] --> API["API Gateway"]
       API --> Agent["에이전트"]
       Agent --> DB["데이터베이스"]
       Agent --> User
   ```

---

### Error 3: HITL (Human-in-the-loop) 슬라이드 누락

**상황:**
- 슬라이드 12장 완성 후 QA 점검 시 "사람 개입 지점" 슬라이드 없음
- 발표 시 청중 질문: "결국 사람이 어디에 개입해?"

**대응:**
1. "에이전트 발표에서 가장 자주 받는 질문이 '사람은 어디에?' 입니다"로 설명
2. HITL 슬라이드 1장 추가 권고:
   ```
   - 에이전트 출력 → 사람 검토
   - 검토 후 승인/거부
   - 거부 시 → 에이전트 수정 또는 에스컬레이션
   ```

## 3) Edge Cases

| # | 쿼리 | 판정 | 이유 |
|---|------|------|------|
| E1 | "발표자료를 너무 많은 데이터로 채웠어. PM 팀 대상인데 기술 정보도 빠뜨리고 싶지 않고. 어떻게 균형을 맞춰야 할까?" | ⚠️ 경계 | PM 팀 대상이면 "워크플로우 + HITL + 로드맵"에 집중. 기술 깊이는 "별도 기술 문서 링크"로 분리 제안. 1메시지/장 원칙 위반 주의 |
| E2 | "C-Level인데 기술 배경이 없어. 아키텍처 다이어그램을 넣으면 이해 못할까?" | ✅ Trigger | C-Level 대상이면 다이어그램 대신 "문제 → 솔루션" 플로우를 시각화. 아키텍처는 "엔지니어링 팀" 슬라이드에 분리 |
| E3 | "이미 기존 도구가 있는데, 우리 에이전트가 그거보다 낫다는 걸 설명해야 해. 경쟁사 비교 슬라이드를 넣어도 될까?" | ⚠️ 경계 | 경쟁사 비교는 위험 요소 많음. "우리 가치 = X" 방식이 낫지, "경쟁사 = 못함"은 법적/윤리적 위험. "차별화 포인트" 슬라이드로 긍정적 메시지 |
| E4 | "배포 후 A/B 테스트를 할 거니까, 그 계획을 슬라이드에 넣으면 좋을까?" | ✅ Trigger | "실험 설계"는 로드맵의 일부로 포함 가능. 특히 "이 에이전트가 정말 효과 있는지 검증 중"을 보여주면 신뢰도 향상 |
| E5 | "투자자 대상 발표인데, 슬라이드 20장이 되었어. 좀 줄여야 할까?" | ✅ Trigger | 투자자 대상은 일반적으로 10~15장 권장. 20장은 긴 편. "핵심 3개 메시지"를 명확히 한 후 재구성 권고 |

## 4) With/Without Skill 비교

### 시나리오: CEO에게 AI 에이전트 도입 승인받기

**Without PPTX AI Slide Skill:**
- 팀이 200장 기술 문서를 PPT로 그냥 옮김
- CEO가 슬라이드 30장까지 보다가 "핵심이 뭐예요?" 질문
- 메시지가 분산되고 비용/ROI 수치가 어디 있는지 찾기 어려움
- **결과: 승인 지연, CEO 신뢰도 하락**

**With PPTX AI Slide Skill:**
- Step 1에서 청중(CEO) + 핵심 메시지(비용 절감 $500/월) + 분량(5-7장) 결정
- Step 2에서 피치 덱 구조 설계: 문제 → 솔루션 → 아키텍처(다이어그램) → 데모 → 비용ROI → HITL → 로드맵
- Step 4에서 PPTX 생성 + 1메시지/장 원칙 적용
- **결과: CEO가 5장만에 핵심 파악 → 즉시 승인**
