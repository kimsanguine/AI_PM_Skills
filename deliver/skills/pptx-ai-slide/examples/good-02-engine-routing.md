# Good Example — 엔진 라우팅 결정 (v0.7.0+)

> 입력 신호를 읽고 4엔진 중 하나로 명확히 라우팅하는 패턴.

---

## Case A — 강의 대량 생산

**요청:**
> "AI Agent 강의 3주차 40장 강의자료 만들어. McKinsey 톤으로."

**라우팅 결정:**
- 장수 40 ≥ 30 → mckinsey
- 명시적 "McKinsey 톤" 신호도 일치

**Step 0 출력:**
```
엔진: mckinsey
근거: 40장 + 강의 시리즈 + McKinsey 톤
체인: 단일 엔진 (체이닝 불필요)
```

---

## Case B — 투자 피치 이미지 강조

**요청:**
> "Series A 5장 피치덱 만들어. 슬라이드마다 어울리는 일러스트 자동 생성해줘."

**라우팅 결정:**
- 장수 5 ≤ 10 + 이미지 생성 요구 → hifidelity

**Step 0 출력:**
```
엔진: hifidelity
근거: 5장 + 슬라이드별 이미지 자동 생성 필수
체인: 단일 엔진
비용 경고: Imagen 호출 5회 → cogs-sentinel 사전 검증 권장
```

## Case C — 디폴트 케이스

**요청:**
> "RAG 아키텍처 15장 미니멀 스타일로."

**라우팅 결정:**
- 장수 15 (5~25 범위) + 미니멀 테마 → html-qa

**Step 0 출력:**
```
엔진: html-qa
근거: 15장 + 미니멀 테마 (디폴트 엔진의 스윗스팟)
체인: 단일 엔진
QA: 100점 루브릭 자동 적용
```

## Case D — 영상 입력 (필수 체이닝)

**요청:**
> "https://youtu.be/abc123 이 강의를 슬라이드로 정리해. 30장 정도."

**라우팅 결정:**
- 영상 입력 → video (전처리 필수)
- 후속: 장수 30 → mckinsey

**Step 0 출력:**
```
엔진: video → mckinsey
근거: YouTube URL + 30장 강의자료
체인:
  1. video: 전사 + 섹션화 + 슬라이드 brief 생성
  2. mckinsey: brief → 30장 PPTX
```

## Case E — PPTX 리디자인 (no-delete 제약)

**요청:**
> "이 기존 강의자료.pptx를 에디토리얼 스타일로 리디자인. 슬라이드 삭제 금지."

**라우팅 결정:**
- 리디자인 + no-delete + 에디토리얼 → html-qa (CSS 스왑 가능한 유일 엔진)

**Step 0 출력:**
```
엔진: html-qa
근거: 리디자인 + no-delete 제약은 HTML CSS 스왑만 안전
체인: 단일 엔진
```
