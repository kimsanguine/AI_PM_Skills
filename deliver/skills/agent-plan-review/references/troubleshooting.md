# Troubleshooting — Agent Plan Review

## 1) Step 0 범위 점검을 건너뛰려는 사용자

- **증상**: "바로 리뷰해줘", "모든 걸 점검해줄 수 있지 않아?" 요청
- **확인**: Step 0에 필수 3개 질문(기존 자산 재사용, MVA, 복잡도 경고)에 응답이 없는가?
- **조치**:
  1. "Step 0 범위 점검은 스킬의 가장 중요한 관문입니다"로 명시
  2. 3가지 질문에 Yes/No 답변 강요 — 이 없이 진행 불가
  3. 명확한 응답 후에만 "범위 옵션 3가지(축소/전체/압축)" 제시

---

## 2) 다이어그램 누락 또는 오래된 상태 유지

- **증상**: 아키텍처 리뷰 섹션에 다이어그램이 없거나 설계 변경 후 업데이트하지 않음
- **확인**:
  - 섹션 1 (아키텍처)에 Mermaid 다이어그램 (flowchart/sequenceDiagram) 2개 이상이 있는가?
  - 리뷰 중 설계 변경이 발생했는데 관련 다이어그램을 업데이트했는가?
- **조치**:
  1. 각 섹션 진입 시 "이 섹션에서 필요한 다이어그램: [이름]"을 명시
  2. 설계 변경이 발생하면 즉시 "이 변경이 [다이어그램명]에 영향을 미칩니다. 함께 업데이트합시다" 안내
  3. 완료 요약에서 "다이어그램 N개 작성, M개 업데이트 필요" 숫자 기록

---

## 3) 장애 모드 매트릭스에서 "치명적 gap" 미인식

- **증상**: 매트릭스 작성 후 "높음/매우높음" 영향도인데 설계에 반영 안 된(❌) 항목을 "차후 개선"으로 넘어감
- **확인**:
  - 매트릭스의 "설계에 반영?" 컬럼에 ❌가 있는가?
  - 그 항목의 영향도가 "높음" 이상인가?
- **조치**:
  1. 해당 항목을 **"치명적 gap"**으로 강조 (예: "⚠️ 치명적 gap: 데이터 누출 위험 미반영")
  2. "구현 전에 반드시 해결 필요"를 의견과 함께 제시
  3. 사용자 선택지: A) 지금 설계 수정 B) 범위 축소하고 v2에서 처리 C) 위험 인정 후 진행 (권고 안 함)

---

## 4) 미결정 사항을 조용히 기본값으로 처리

- **증상**: 사용자가 "그냥 계속해"라고 요청했을 때, 미결 사항들을 조용히 기본 가정으로 설정하고 진행
- **확인**:
  - AskUserQuestion에 대한 명확한 응답(A/B/C 선택)이 있었는가?
  - 응답 없이 "계속 진행" 요청이 있었는가?
- **조치**:
  1. **절대 조용히 기본 설정하지 말 것**
  2. 미결 항목을 명시적으로 기록하는 "미결정 사항" 섹션 작성
  3. 각 미결 항목에 "가정한 방향" + "위험도" + "구현 전 합의 필요 여부" 기록
  4. 완료 요약에서 "미결정 사항: N개" 수를 명시

---

## 5) 이슈 번호 또는 옵션 레이블 누락

- **증상**: AskUserQuestion을 제시했지만 사용자가 응답할 때 "어느 이슈의 어느 옵션을 선택했는지" 애매함
- **확인**: 각 이슈에 번호(1, 2, 3...)가 있고, 옵션에 문자(A, B, C...)가 있는가?
- **조치**:
  1. 모든 이슈에 번호 명시: "**이슈 2:** ..."
  2. 모든 옵션에 문자 명시: "A) ...", "B) ...", "C) ..."
  3. AskUserQuestion에서 "이슈 2A를 권고합니다" 형식으로 참조 가능하게 구성

---

## 6) 컨텍스트 부족으로 인한 일반적 조언 제시

- **증상**: 리뷰 중 "프롬프트를 더 구체적으로 작성하세요", "아키텍처를 단순화하세요" 같은 추상적 권고만 제시
- **확인**: 각 권고안이 설계 문서의 특정 부분을 지적하고 있는가? 구체적 대안이 있는가?
- **조치**:
  1. 모든 권고는 "설계의 어느 부분이 문제인지" 명시: "Section 2의 Anti-Goals에 ..."
  2. 최소 2-3가지 옵션(A/B/C) 제시 + 각각의 비용/위험도/노력 한 줄 설명
  3. 권고 옵션(예: A) 부터 시작하는 것이 일반적이므로, 권고 옵션을 항상 먼저 나열
