# Test Cases — infographic-gif-creator 스킬

## 1) Trigger Tests

### Should Trigger (5)

1. "3-Tier 에이전트 아키텍처를 설명하는 GIF를 만들어 줄 수 있어? 사용자 입력이 Orchestrator를 거쳐 3개 Sub-agent로 분기되는 과정을 단계별로 보여줘."
   - 이유: 시각화 대상(3-Tier 구조), 포맷(GIF), 애니메이션 방식(단계별) 명확 → Step 1 장면 설계 진입 가능

2. "우리 워크플로우의 데이터 흐름을 1분짜리 루프 GIF로 만들어 줄 수 있어요? Slack → API → Database 순으로 흘러가는 걸 보여줘야 해."
   - 이유: 목표(데이터 흐름 시각화), 길이(1분), 루프 여부(루프형) 명시 → Step 1 메시지 정의부터 시작

3. "How do I visualize a multi-step workflow with animation in GIF format? We need something for README documentation."
   - 이유: 용도(문서), 포맷(GIF), 내용(멀티스텝) 제시 → Step 2 HTML/CSS 구현 단계로 진입 가능

4. "모델별 비용 비교를 애니메이션으로 보여주는 GIF를 만들어 줄 수 있어? Haiku → Sonnet → Opus로 갈수록 바 차트가 올라가는 식으로."
   - 이유: 시각화 유형(비용 비교), 애니메이션 효과(바 차트 애니메이션) 명확 → 커스텀 데이터 바인딩 설계

5. "에이전트 에러 처리 플로우를 6초짜리 GIF로 만들어 줄 수 있어? 정상 흐름 → 에러 감지 → 폴백 처리 순서로."
   - 이유: 흐름(3단계), 길이(6초), 목적(에러 시각화) 명시 → Step 1 시간 배분 설계 가능

### Should NOT Trigger (5)

1. "에이전트 아키텍처 전체를 설명하는 영상을 만들어 줄 수 있어? 5분짜리로."
   - 올바른 라우팅: 멀티씬 장편 영상 → `agent-demo-video` 또는 `compose-video`

2. "우리 사무실 사진을 넣어서 회사 소개 슬라이드쇼를 만들어 줄 수 있어?"
   - 올바른 라우팅: 정적 슬라이드 → `pptx-ai-slide` 또는 `gemini-image-flow`

3. "AI 생성 이미지를 애니메이션으로 움직이게 만들 수 있어요?"
   - 올바른 라우팅: 일반 이미지 애니메이션 → 비전문 도구 또는 Adobe After Effects (스킬 범위 밖)

4. "데이터 분석 결과를 인터랙티브 대시보드로 보여주고 싶어요."
   - 올바른 라우팅: 인터랙티브 대시보드 → Tableau/Metabase (애니메이션 GIF 아님)

5. "정적 인포그래픽 하나만 필요해요. GIF가 아니라 PNG나 SVG로."
   - 올바른 라우팅: 정적 이미지 생성 → `gemini-image-flow`

## 2) Edge Cases

### 경계 사례 (5)

1. **Puppeteer 버전이 19.0 미만일 때 CDP beginFrame 사용 불가**
   - 입력: "Puppeteer를 설치했는데 버전이 18.0이래요. beginFrame은?"
   - 예상 행동: 버전 확인 → Puppeteer ≥19.0으로 업그레이드 또는 CSS animation-play-state 방식으로 대안 구현
   - 근거: CDP beginFrame은 Puppeteer 19.0+ 지원 → 낮은 버전에서는 의사결정 트리(Step 3)에서 대안 선택 필수

2. **텍스트가 너무 많아서 노출 시간 부족한 경우**
   - 입력: "300글자의 설명을 GIF에 넣었는데 2초 만에 지나가요. 어떻게 해요?"
   - 예상 행동: 텍스트 시간 공식 적용 → `(300 ÷ 12) + 1 = 26초 필요` → 전체 GIF 길이 연장 또는 텍스트 압축(100글자로 재작성)
   - 근거: 텍스트 가독성 = (글자 수 ÷ 12) + 1초 기준 → 이 공식보다 짧으면 품질 저하

3. **GIF 파일이 8MB를 초과할 때 압축 고민**
   - 입력: "GIF 렌더링 결과가 12MB예요. 8MB 이하로 줄일 수 있을까요?"
   - 예상 행동: 단계적 최적화 (Step 4 도메인 파일 참고): fps 낮추기 → 해상도 축소 → 색상 수 제한 → gifsicle lossy
   - 근거: GIF는 포맷 자체가 무거움 → 다단계 최적화로 15-30% 추가 감소 가능

4. **캡처 방법 의사결정 트리에서 모든 조건이 "No"일 때**
   - 입력: "Puppeteer도 19.0 이상이 아니고, CSS animation-play-state 제어도 못 하는데 어떻게 해요?"
   - 예상 행동: 의사결정 트리 최후 수단(animationend 이벤트 기반) 사용 → 타이밍 오차 ±2-3프레임 감수 → 혹은 해상도/fps 낮춰서 프레임 드롭 최소화
   - 근거: 모든 방법이 부족하면 화질 or 정확도 중 하나 sacrifice 필수

5. **루프형 GIF에서 시작/끝 프레임 시각차가 5%를 초과할 때**
   - 입력: "루프형 GIF를 만들었는데 시작과 끝이 너무 달라서 끊어져 보여요."
   - 예상 행동: Step 5에서 시각차 재확인 → 시작/끝 프레임 색상/위치/텍스트 일치하도록 CSS 재조정 → 재캡처
   - 근거: 루프 연속성 = 사용자 경험 핵심 → 매끄러운 루프를 위해 프레임 마지막 1초에서 미리 시작 프레임처럼 돌아가야 함
