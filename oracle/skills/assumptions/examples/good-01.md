# Good Example — assumptions

## 사용자 요청
"고객 문의를 자동 분류하는 에이전트를 만들려고 하는데, 이 아이디어의 핵심 가정들을 분석해줘. LLM이 우리 도메인 용어를 이해할 수 있을지, 고객이 자동 응답을 수용할지 확신이 없어."

## 승인 이유
- 구현 전 에이전트의 숨겨진 가정을 발굴하고 리스크를 수치화하려는 요청
- 4축(Value/Feasibility/Reliability/Ethics) 검증이 필요한 에이전트 기회가 이미 선택된 상태
- 팀이 "정말 작동할 것 같나"에 대한 구체적인 우려(도메인 용어 이해, 고객 수용성)를 표현

## 예상 처리
1. 에이전트 개요 확인 (고객 문의 분류, 자동 응답 포함)
2. 4축 가정 브레인스토밍 (최소 3개 이상 per axis)
   - Value: "고객이 자동 응답을 실제로 사용하는가?"
   - Feasibility: "우리 도메인 데이터에 LLM이 접근 가능한가?"
   - Reliability: "분류 정확도가 충분한가? (90%? 95%?)"
   - Ethics: "자동화해도 고객이 불쾌하지 않을까?"
3. 우선순위 점수 계산 및 Top 5 기준으로 정렬
4. Top 3 가정 심층 분석 (틀렸을 시 시나리오 + 피벗 옵션)
5. 2일 이내 가능한 검증 실험 설계
6. Ethics 위험 평가 및 hitl 스킬 연결 검토
