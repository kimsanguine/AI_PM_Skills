# Bad Example — assumptions

## 사용자 요청
"이번 달 에이전트 토큰 비용 리뷰해줘. 예산 대비 초과된 것 같아."

## 거절 이유
- 기존에 이미 실행 중인 에이전트의 비용 추적/최적화 요청 (실시간 모니터링)
- assumptions 스킬은 **구현 전 가정 검증**에 특화되어 있음
- 이미 배포된 에이전트의 비용 리뷰는 다른 스킬(burn-rate) 범위

## 올바른 라우팅
- 실시간 비용 추적/최적화: `argus/burn-rate`
- 새로운 에이전트 구현 전 비용 예측: `oracle/cost-sim`

## 수정 방향
- "새로운 에이전트를 구축하려고 하는데, 예상 비용과 ROI를 검증하고 싶어. 이 아이디어가 비용 효율적일지 확신이 없어." → assumptions 스킬로 라우팅 가능
