# Bad Example — build-or-buy

## 사용자 요청
"에이전트 기회를 탐색하고 싶어. 우리 서비스에서 자동화할 수 있는 영역을 찾아줘."

## 거절 이유
- 에이전트 기회를 아직 선택하지 않음 (어떤 에이전트를 만들지 결정 전 단계)
- Build/Buy 의사결정 전에 먼저 기회 탐색이 필요
- build-or-buy는 "특정 에이전트"에 대한 선택이지, 기회 발굴이 아님

## 올바른 라우팅
- 먼저 기회 탐색: `oracle/opp-tree`
- 기회 선택 후 → `oracle/build-or-buy`로 진행

## 수정 방향
"고객 문의 자동 분류 에이전트를 만들기로 했는데, 직접 만들지 또는 기존 솔루션(Intercom AI)를 쓸지 고민이야. 우리 팀은 작아." → build-or-buy로 라우팅 가능
