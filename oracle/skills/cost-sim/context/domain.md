# Domain Context — cost-sim

## 1) Domain Scope

Agent Operating Cost Simulation — 에이전트 운영 비용을 사전에 모델링하고 예측하는 도메인

**적용 범위:**
- 신규 에이전트의 예상 비용 추정 (구현 전)
- 스케일 시나리오별 비용 곡선 예측 (1명 → 100명 → 1000명)
- 모델/API 선택의 비용 최적화
- 비용 KPI 설정 및 예산 결정

**제외 범위:**
- 실시간 비용 추적/모니터링 (burn-rate 스킬)
- 이미 배포된 에이전트의 성능 최적화 (별도 스킬)
- 비용 관계없이 기술 성능 기준만 필요한 경우 (router 스킬)

---

## 2) Primary Users

- **PM / Product Manager**: 에이전트 구현 전 비용-편익 분석
- **Finance / Budget Owner**: 월간/연간 비용 예측으로 예산 계획
- **Tech Lead**: 모델 선택(Haiku vs Sonnet vs Opus)의 비용 영향 분석
- **Sales / GTM**: 고객 세그먼트별 가격 책정 근거

---

## 3) Required Inputs

- **에이전트 유형**: Cron Agent, Monitor Agent, On-demand Agent, Orchestrator 등
- **모델 선택**: Claude Haiku/Sonnet/Opus, GPT-4o/4o-mini 등
- **호출 빈도**: 일간/월간/연간 기준으로 트리거 횟수
- **회당 입출력**: 평균 입력 토큰, 평균 출력 토큰 (또는 예상 범위)
- **외부 API 사용**: Brave Search, Google Search, SendGrid 등

---

## 4) Output Contract

**Step-by-Step Deliverables:**

1. **비용 시뮬레이션 템플릿** (완전히 작성됨)
   - 토큰 비용 상세 계산
   - 외부 API 비용 추가
   - 월간/연간 총 비용

2. **스케일 시나리오 분석** (3단계)
   - 사용자 1명: $X/월
   - 사용자 10명: $X/월
   - 사용자 100명: $X/월
   - 비용 폭발 지점 표시

3. **최적화 전략** (최소 2개 이상)
   - 모델 라우팅 (비용 절감 %)
   - 캐싱 전략 (감소 효과)
   - 입력 최적화 (절감률)
   - 예상 절감 비용

4. **비용 KPI 권장값**
   - 월간 비용 상한
   - 비용 폭발 감지 임계값

5. **다음 단계 연결**
   - 비용 수용 가능 → instruction-design
   - 비용 과다 → build-or-buy 재검토

---

## 5) Guardrails

**필수 체크사항:**

- ✓ 모든 단가에 "2026-03 기준" 명시
  - 모델 가격은 빠르게 변동함을 강조
  - 공식 가격 페이지 링크 제공

- ✓ 추정치를 확정 수치로 표현하지 않음
  - "$1,200/월 정확" (X)
  - "$1,000~$1,500/월 범위" (O)

- ✓ 외부 API 무료 티어 한도 확인
  - "Google Search API 100쿼리/일 무료" 명시
  - 초과 비용 명확히 표시

- ✓ 스케일 시나리오에서 비선형 증가 경고
  - "100명 규모에서 비용이 1명의 200배 → 오케스트레이터 폭증 가능"

## 6) Working Facts

**에이전트 비용 모델링의 일반적 특성:**

| 항목 | 특징 | 참고 |
|------|------|------|
| **비용 구조** | 고정비용 거의 없음, 사용량 비례 | 일반 SaaS와 완전히 다름 |
| **모델별 차이** | Haiku vs Sonnet = 12배 비용 차이 | 작업 복잡도에 따라 선택 필수 |
| **스케일링** | 사용자 10배 증가 = 비용 10배 증가 (기본) | 오케스트레이터는 초과 비례 (주의) |
| **외부 API** | 무료 티어 + 초과 비용 구조 | 예측 어려움 → 보수적 추정 권장 |
| **배치 처리** | Batch API = 50% 할인 | 긴급하지 않은 작업은 배치 권장 |
