# Good Example — opp-tree

## 사용자 요청
"우리 CS팀에서 반복적으로 하는 업무가 많은데, 어떤 부분을 에이전트로 자동화하면 좋을지 기회를 체계적으로 분석해줘. 현재 티켓 분류, FAQ 응답, 에스컬레이션 판단을 수동으로 하고 있어."

## 승인 이유
- 에이전트 기회를 아직 선택하지 않음 (여러 선택지 중 최고 우선순위 찾기)
- 구체적인 반복 업무(티켓 분류, FAQ, 에스컬레이션)가 명시됨
- 자동화 기회를 체계적으로 우선순위 매기고 검증 실험 설계 필요

## 예상 처리
1. Automation Outcome 확정 (CS팀의 시간 단축 구체화)
2. Opportunities 발굴 (3~5개 기회 추출, 점수 계산)
3. Top 3 선택 및 Agent Solutions 후보 (각 기회마다 2~3개 에이전트 유형)
4. 최우선 솔루션에 대한 4축 가정 정의 (Value/Feasibility/Reliability/Ethics)
5. 검증 실험 설계 (2일 이내 가능)
6. 다음 단계 (assumptions 스킬로 연결 또는 build-or-buy)
