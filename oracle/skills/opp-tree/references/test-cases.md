# Test Cases — opp-tree 스킬

## 1) Trigger Tests

### Should Trigger (5)

1. "우리 회사의 반복 업무를 AI 에이전트로 자동화하고 싶은데 어디부터 시작할까요?"
   - 이유: 자동화 기회 탐색 요청 명확 → Step 1 Automation Outcome 정의부터 시작

2. "여러 에이전트 아이디어가 있는데 어느 것을 먼저 구현할지 우선순위를 정하고 싶어요."
   - 이유: 기회 우선순위 결정 필요 → Step 2 Opportunity Score 계산으로 진입

3. "고객 지원팀의 반복 작업을 분석해서 자동화 가능한 것들을 찾아 줄 수 있어요?"
   - 이유: 도메인 특화 기회 탐색 → Opportunity 발굴 (Step 2) 진입

4. "선택한 자동화 기회를 정말 구현해야 하는지 검증해 주세요. 어떤 실험을 해야 할까요?"
   - 이유: 검증 실험 설계 요청 → Step 4 Experiments 설계 진입

5. "기술 팀과 비즈니스 팀의 의견이 갈려요. 데이터 기반으로 '이 에이전트를 만들어야 하는가'를 판단하고 싶어요."
   - 이유: 의사결정 데이터화 → AOT 프레임워크 전체 설계 (Step 1~5)

### Should NOT Trigger (5)

1. "선택한 에이전트의 프롬프트와 인스트럭션을 작성해 줄 수 있어요?"
   - 올바른 라우팅: AOT 완료 후 구현 → `forge/instruction` 또는 `agent-plan-review`

2. "이 에이전트의 기술 스택(Haiku vs Sonnet)을 결정해 줄 수 있어요?"
   - 올바른 라우팅: 모델/비용 선택 → `atlas/router` 또는 `oracle/cost-sim`

3. "우리 전체 비즈니스 전략을 분석해 줄 수 있어요?"
   - 올바른 라우팅: 경영 전략 분석 → AOT 범위 밖 (에이전트 기회만 다룸)

4. "어떤 에이전트가 기술적으로 구현 가능한지 평가해 줄 수 있어요?"
   - 올바른 라우팅: 기술 타당성 검증 → `build-or-buy` (아키텍처 선택)

5. "경쟁사들은 어떤 에이전트를 쓰고 있나요?"
   - 올바른 라우팅: 시장 조사 → AOT 범위 밖

## 2) Edge Cases

### 경계 사례 (5)

1. **모든 Opportunity의 점수가 높게 나올 때 (3점 이상 × 5개+)**
   - 입력: "계산했는데 모든 기회가 좋아 보여요. 이게 맞나요?"
   - 예상 행동: 점수 계산 재검토 → 팀과 "정말 자동화 가능한가" 논의 → 판단 의존도 재평가 (예: 점수 높게 평가 바이어스)
   - 근거: 모든 기회가 높은 점수 = 평가 오류 가능성 높음 → 재검토 필수

2. **선택한 기회가 실제로 반복되지 않을 때 (실험 후 판명)**
   - 입력: "선택한 기회를 2주 실험했는데 일주일에 1번도 안 반복되네요."
   - 예상 행동: 해당 기회 제외 → 다음 순위 기회로 즉시 전환 (재점수 불필요)
   - 근거: 반복 빈도 오판 = Opportunity 점수 부정확 → 실제 데이터 기반 조정

3. **필요 API/도구가 실제로 없을 때 (Phase 4 Feasibility 검증 실패)**
   - 입력: "선택한 기회를 구현하려고 API를 찾았는데 권한이 없거나 비용이 $5k/month 이상이에요."
   - 예상 행동: Feasibility 점수 재평가 → 낮은 점수 기회로 전환 또는 기술 솔루션 재검토
   - 근거: Feasibility 검증 미흡 = 프로젝트 실패 위험 → 실험 단계에서 조기 발견 필수

4. **Ethics 위험이 높은데 AOT에서 미감지된 경우**
   - 입력: "자동화하기 좋아 보였는데 배포 후 '이건 자동화하면 안 된다'는 지적이 나왔어요."
   - 예상 행동: 다음 AOT부터 Ethics 축 리뷰 미팅 추가 → hitl 스킬과 함께 HITL 재설계 후 배포
   - 근거: Ethics = 사후 조정 불가능 → 설계 단계에서 팀 리뷰 필수

5. **Opportunity Score가 낮은데 구현을 강력히 원할 때**
   - 입력: "점수는 낮은데 CEO가 꼭 이 에이전트를 만들어야 한다고 해요."
   - 예상 행동: 점수와 의사결정 gap 문서화 → "비즈니스 우선순위로 선택하되, 리스크 인식"을 명시 → 실험 기간 연장 (데이터 기반 재평가)
   - 근거: 점수 기반 선택과 비즈니스 우선순위는 별개 → 선택 근거 투명화 필수
