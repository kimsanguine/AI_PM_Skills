## Agent KPI — 4축 성과 지표

에이전트 성과를 측정하는 4가지 차원:

```
        정확도 (Accuracy)
             ↑
비용 ←───────┼───────→ 신뢰성
(Cost)       │        (Reliability)
             ↓
        레이턴시 (Latency)
```

4개 축은 트레이드오프 관계입니다:
- 정확도↑ → 비용↑ (더 좋은 모델)
- 레이턴시↓ → 비용↑ (더 빠른 처리)
- 신뢰성↑ → 개발 비용↑ (에러 처리 강화)

목표는 **비즈니스 요구에 맞는 최적 균형점** 찾기입니다.

---

### Axis 1 — Accuracy (정확도)

**정의:** 에이전트 출력이 의도한 기준을 충족하는 비율

측정 방법:
```
자동 평가: 출력 형식 검증 (JSON 파싱, 필수 필드 존재 여부)
샘플 평가: N건 중 사람이 직접 품질 평가
A/B 평가: 두 버전 출력 비교
```

KPI 설정 기준:
| 에이전트 유형 | 권장 정확도 목표 |
|---|---|
| 뉴스 요약/브리핑 | 90%+ (오류 허용 범위 넓음) |
| 이메일 분류/필터링 | 95%+ (중요 이메일 누락 위험) |
| 데이터 처리/변환 | 99%+ (오류가 다운스트림에 영향) |
| 의사결정 지원 | 인간 검토 필수 (정확도 목표보다 투명성 중요) |

---

### Axis 2 — Cost (비용)

**정의:** 에이전트 운영에 소요되는 비용 (API + 인프라)

핵심 지표:
```
Cost per Run = 입력 토큰 × 단가 + 출력 토큰 × 단가
Monthly Cost = Cost per Run × 월 실행 횟수
Cost per Outcome = Monthly Cost ÷ 월 아웃컴 수
```

비용 목표 설정 방법:
```
Step 1: 현재 비용 측정 (2주 모니터링)
Step 2: ROI 계산 (에이전트가 절감하는 가치 vs 비용)
Step 3: 허용 비용 상한 설정 (ROI > 3x 권장)
Step 4: 비용 초과 시 알림 설정
```

---

### Axis 3 — Latency (레이턴시)

**정의:** 트리거부터 출력 완료까지 소요 시간

측정 포인트:
```
E2E Latency = API 응답 시간 + 처리 시간 + 전달 시간

크론 에이전트: 스케줄 시간 기준 ±N분 내 완료
대화형 에이전트: 응답 시간 N초 이내
파이프라인 에이전트: 전체 파이프라인 완료 시간
```

레이턴시 목표 기준:
| 에이전트 유형 | 허용 레이턴시 |
|---|---|
| 실시간 대화 | < 3초 |
| 백그라운드 분석 | < 2분 |
| 일일 크론 브리핑 | < 10분 |
| 주간 리포트 | < 30분 |

타임아웃 = 레이턴시 목표 × 3 으로 설정 권장

---

### Axis 4 — Reliability (신뢰성)

**정의:** 에이전트가 의도한 대로 일관되게 실행되는 비율

핵심 지표:
```
Success Rate = 성공 실행 수 ÷ 전체 실행 수 × 100%
MTTR = 장애 발생 후 복구까지 평균 시간
Error Rate by Type = 유형별 오류 발생 비율
```

신뢰성 등급:
| 등급 | 성공률 | 적합 에이전트 |
|---|---|---|
| Basic | 90%+ | 내부 편의 도구 |
| Standard | 95%+ | 일반 업무 자동화 |
| High | 99%+ | 비즈니스 핵심 프로세스 |
| Critical | 99.9%+ | 금융/의료/규제 환경 |

### Instructions

You are helping design a KPI framework for: **$ARGUMENTS**

**Step 1** — 에이전트 유형 파악 (브리핑/분류/파이프라인/의사결정 지원)

**Step 2** — 4축 KPI 목표값 설정  
각 축의 비즈니스 맥락에 맞는 목표값 결정

**Step 3** — 측정 방법 정의  
자동/수동 평가 방법, 측정 주기, 도구

**Step 4** — 대시보드 구성  
4축 KPI를 시각화할 모니터링 구조 설계

**Step 5** — 알림 임계값 설정  
각 KPI의 경고/위험 임계값과 알림 채널

**Step 6** — 다음 연결  
OKR과 연계: `/agent-okr`
