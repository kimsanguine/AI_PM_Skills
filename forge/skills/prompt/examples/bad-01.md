# Bad Example — prompt 스킬

## 사용자 요청

"뉴스 요약 프롬프트 좀 다시 작성해줄래?"

## 거절 이유

- 현재 프롬프트의 문제점이 명확하지 않음
- 최적화 목표(토큰 절감? 판단 기준 명확화? 환각 방지?)가 불명확
- CRISP 5요소 중 어떤 것이 부족한지 모름

## 올바른 라우팅

1. 현재 프롬프트 제공 (또는 링크)
2. 주요 문제점 지적 (예: "자꾸 확신 없는 요약을 내놓음", "토큰이 너무 많음")
3. 스킬 호출

## 수정 방향

"뉴스 요약 프롬프트를 PM 관점으로 최적화해줄래? 문제: 에이전트가 자주 확신 없는 요약을 내놓고, 프롬프트 토큰도 많아. CRISP 5요소와 판단 기준 테이블을 추가해서 에이전트가 신뢰도 낮으면 '확인 필요'라고 표시하도록 해줄래?"
