# Good Example — prompt 스킬

## 사용자 요청

"뉴스 요약 프롬프트를 PM 관점으로 최적화해줄래? 지금은 토큰만 많고 에이전트가 자주 확신 없는 요약을 내놓아."

## 승인 이유

- 프롬프트가 의도 명확화 부족
- CRISP 5요소(Context/Role/Instruction/Scope/Parameters) 중 일부 누락
- 판단 기준이 정성적이어서 에이전트가 해석에 헷갈림
- Why-First 원칙 미적용

## 예상 처리

1. Context: "당신은 PM 담당자의 정보 수집 파트너입니다. 매일 아침 자동 실행되어, 전 날 AI/에이전트 뉴스를 전달합니다."
2. Role: "뉴스 큐레이터(PM 관점)로서 단순 요약이 아닌 PM 인사이트 포함"
3. Instruction: "3줄 이내 요약 → 왜: PM이 빠르게 의사결정 필요 정보 얻기 위해"
4. Scope: "포함: AI/에이전트 관련 뉴스, 제외: 스팸/학술 논문만"
5. Parameters: "형식: Markdown, 길이: 3줄, 언어: 한국어, 톤: 간결하고 PM 친화적"
6. 판단 기준 테이블: 신뢰도 높음/낮음별 행동 정의
7. 실패 처리: "신뢰도 낮음 → '❓ 확인 필요' 표시", "뉴스 5개 미만 → 있는 것만 처리"

## 최종 결과물

의도가 명확한 프롬프트로 에이전트가 예외 상황에서도 올바르게 판단할 수 있음
