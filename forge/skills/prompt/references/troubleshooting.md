# Troubleshooting — prompt 스킬

## 1) CRISP 5요소 중 일부 누락

**증상:**
- 프롬프트에 Role과 Instruction만 있고, Context, Scope, Parameters 미정의
- 에이전트가 언제/어디서 실행되는지 알 수 없음

**확인:**
- Context: "사용자는 누구? 실행 환경은? 접근 가능한 데이터는?"
- Scope: "해야 할 것/하면 안 될 것이 명확한가?"
- Parameters: "출력 형식/길이/언어/톤이 명시되었는가?"

**조치:**
- 누락된 요소 추가 (최소 4개 명시)
- 특히 Scope(포함/제외)는 Anti-Goals와 함께 필수

---

## 2) Anti-Goals 없음 또는 추상적

**증상:**
- Anti-Goals 부재 → 에이전트가 추측도 출력
- Anti-Goals 있지만 추상적: "정확하게", "좋게" → 구체적이지 않음

**확인:**
- 절대 하지 말아야 할 것이 구체적인 시나리오로 작성되었는가?
- "이 상황에서는 이렇게 하면 안 된다"고 판단할 수 있는가?

**조치:**
- Anti-Goals를 구체적 시나리오로 재작성:
  ```
  ❌ "정확하게"
  ✅ "신뢰도 < 0.7인 정보는 '❓ 확인 필요' 표시"
  ```

## 3) 판단 기준이 정성적

**증상:**
- "중요한 정보는 강조해주세요", "좋은 품질로 요약해주세요"
- 에이전트가 해석에 헷갈림

**확인:**
- 판단 기준이 임계값, 점수, 숫자로 표현되었는가?
- "이 점수 이상이면 강조", "이 조건이면 포함/제외" 같은 명확한 기준이 있는가?

**조치:**
- 정량적 기준 재정의:
  ```
  ❌ "중요한 정보"
  ✅ "마감일 포함, CEO 발신, 키워드[계약/결제/긴급] → 항상 강조"
  ```

## 4) 컨텍스트 과부하

**증상:**
- 프롬프트가 너무 길어서 실제 입력 토큰 부족
- 컨텍스트 윈도우의 50% 이상이 시스템 프롬프트 차지

**확인:**
- 프롬프트 길이가 컨텍스트 윈도우의 몇 %를 차지하는가?
- 정말 필요한 정보만 포함되었는가?

**조치:**
- forge/ctx-budget 스킬로 라우팅
- 또는 프롬프트 압축:
  1. 반복 패턴 → SKILL.md로 외부화
  2. 구체적 예시 → 일반화
  3. 주석/설명 → 최소화

## 5) Why-First 원칙 미적용

**증상:**
- 프롬프트: "3줄로 요약해줘", "정확하게 해줘"
- 문제: 에이전트가 "왜" 3줄이어야 하는지 모름
- 예외 상황에서 판단 불가

**확인:**
- 각 주요 지시마다 "왜"가 설명되었는가?

**조치:**
- Why-First 적용:
  ```
  ❌ "3줄 요약해줘"
  ✅ "3줄 이내로 요약해줘. 왜: PM이 빠르게 의사결정에 필요한 정보를 얻기 위해"
  ```

## 6) 실패 처리 누락

**증상:**
- 프롬프트에 성공 경로만 있고, "데이터 없으면?", "불확실하면?", "에러 나면?"의 대응 없음

**확인:**
- 데이터 없을 때의 행동이 정의되었는가?
- 판단이 불확실할 때의 대응이 정의되었는가?
- API 실패 시의 행동이 정의되었는가?

**조치:**
- 실패 케이스별 행동 명시:
  ```
  데이터 없음: "데이터가 없습니다" 메시지 + 종료
  판단 불확실: "❓ 확인 필요" 표시 + 계속 진행
  API 에러: 3회 재시도 → 실패 시 대체값 사용
  ```

## 7) 역할 충돌

**증상:**
- "창의적이면서 정확하게", "빠르면서 완벽하게" 같은 상충하는 요구사항
- 에이전트가 어느 쪽을 우선해야 할지 불명확

**확인:**
- 상충하는 요구사항이 있는가?
- 우선순위가 명확한가?

**조치:**
- 역할 충돌 해결:
  ```
  ❌ "창의적이고 정확하게"
  ✅ "정확성을 최우선으로, 간결하게. 창의적 표현은 그 다음"
  ```
