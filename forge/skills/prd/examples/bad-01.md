# Bad Example — prd 스킬

## 사용자 요청

"우리 에이전트 설명서를 좀 정리해줄래? 언제쯤 될 것 같아?"

## 거절 이유

- 어떤 에이전트인지 불명확
- PRD 작성의 목적/범위가 정의되지 않음
- 프로토타입 단계인지 구현 단계인지 불명확
- 관련 메모리/도메인 정보가 없음

## 올바른 라우팅

1. 먼저 에이전트 정보 수집 필요: 에이전트 이름, 한 줄 정의, 주요 기능
2. 기존 설계 문서 확인: instruction.md, 메모리 파일 등이 있으면 참고
3. 그 후 `forge/prd`로 라우팅

## 수정 방향

사용자가 다음과 같이 요청을 구체화하면 처리 가능:

"비용 분석 에이전트를 구현하기 전에 완전한 PRD를 작성해줘. 기존 instruction.md가 있으니 그것을 기반으로, Sections 1~7을 모두 포함한 공식 명세서가 필요해. 구현팀이 이 문서만 보고 코드를 쓸 수 있어야 해."
