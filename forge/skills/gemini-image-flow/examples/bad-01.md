# Bad Example — gemini-image-flow 스킬

## 사용자 요청

"Gemini로 이미지 만드는 파이프라인을 설계해줄래?"

## 거절 이유

- 어떤 용도의 이미지인지 불명확 (UI? 배너? 로고?)
- 프로토타입 vs 프로덕션 구분 불명확
- 후속 파이프라인(코드 변환?)이 필요한지 모름
- API 환경 준비 상태 불명확

## 올바른 라우팅

1. 이미지 생성 목적 확인 (UI/디자인, 배너, 로고 등)
2. API 키 발급 여부 확인
3. 후속 파이프라인 필요 여부 확인
4. 스킬 호출

## 수정 방향

"Gemini를 이용한 UI 자동 생성 파이프라인을 설계해줄래? 사용자가 프로젝트 설명을 입력하면, Gemini Flash로 이미지 생성하고, 마지막으로 그 이미지를 Claude Code에서 React 코드로 변환하는 sketch-to-code 파이프라인까지. API 키는 이미 있어."
