# Test Cases — agent-demo-video 스킬

## 1) Trigger Tests

### Should Trigger (5)

1. "고객 문의 자동 분류 에이전트의 데모 영상을 만들어줘. 투자자 발표용 60초."
   - 이유: 구체적인 에이전트(자동 분류), 청중(투자자), 목표 길이(60초) 명시 → 즉시 스토리라인 설계 가능

2. "How do I create a demo video for our knowledge base search agent? We need it for our landing page (30 seconds)."
   - 이유: 에이전트 설명(지식 검색), 용도(랜딩페이지), 길이(30초) 명확 → Remotion 구성 설계 진입 가능

3. "에이전트 데모 영상이 필요해. Slack 메시지가 들어오면 자동으로 우선순위 분류하고 담당자에게 배정하는 과정을 보여줘야 해."
   - 이유: 워크플로우(슬랙→분류→배정) 명확, 프로세스 시각화 요청 → Step 1 스토리라인 설계 진입

4. "우리 마케팅 에이전트가 블로그 기사 아이디어를 생성하는 과정을 영상으로 만들어 줄 수 있어? 팀 온보딩용 4분 정도."
   - 이유: 에이전트 기능(기사 생성), 용도(온보딩), 길이(4분) 제시 → 5단계 구조로 설계 가능

5. "Create a 90-second demo video showing how our data validation agent catches errors in spreadsheets. Include before/after comparison."
   - 이유: 에이전트(데이터 검증), 목표(오류 감지), 시각화 방식(Before/After) 명시 → 결과 장면 구현 진입

### Should NOT Trigger (5)

1. "에이전트 영상을 만들어 줄 수 있어?"
   - 올바른 라우팅: 에이전트 설계가 미완료 → `agent-plan-review` (아키텍처 먼저 확정)

2. "우리 웹사이트에 넣을 슬라이드 쇼 디자인을 만들어줄 수 있나?"
   - 올바른 라우팅: 정적 슬라이드 → `pptx-ai-slide` 또는 `gemini-image-flow`

3. "에이전트 아키텍처를 설명하는 GIF를 만들어 줘. 3-Tier 구조를 단계별로 표시하고."
   - 올바른 라우팅: 짧은 애니메이션 GIF → `infographic-gif-creator`

4. "에이전트의 데모 영상을 만들어야 하는데, TTS 나레이션은 직접 준비할 거야. 다만 인코딩만 도와줄 수 있어?"
   - 올바른 라우팅: 최종 인코딩/FFmpeg 처리만 필요 → 외부 도구 또는 `compose-video` 기술 스킬

5. "우리 랜딩 페이지에 실제 제품 스크린샷과 데모 영상이 섞여 있는 동영상을 만들어줄 수 있어?"
   - 올바른 라우팅: 멀티씬 최종 렌더링 → `compose-video` (여러 에셋을 병렬/순차로 합성)

## 2) Edge Cases

### 경계 사례 (5)

1. **에이전트 워크플로우가 길수록 스토리라인이 어려워지는 경우**
   - 입력: "우리 에이전트는 10단계를 거칩니다. 모두 보여줄 수 있어요?"
   - 예상 행동: 모든 단계를 보여주는 대신, Hook/Problem/Solution/Result 5단계 구조에 "가장 중요한 3-4 단계"만 선별해서 스토리라인 재설계
   - 근거: 영상 길이 제약 + 청중의 주의력 한계 → 핵심 가치만 강조하는 압축 스토리가 필수

2. **데이터 바인딩 필드가 동적으로 변할 때 프레임 정확도 손상**
   - 입력: "에이전트 결과가 매번 다른데(생성된 텍스트, 이미지), 이걸 영상에 어떻게 반영하지?"
   - 예상 행동: null-safe 처리 + 샘플 데이터로 Composition 검증 후 렌더 → 실제 배포 시 인자 치환 가능하도록 설계
   - 근거: 재현 가능한 렌더링 = Remotion의 핵심 가치 → 프롭스 구조를 미리 정의해야 반복 재사용 가능

3. **화면 녹화 에셋이 없거나 부실한 경우**
   - 입력: "에이전트 동작 영상 예제가 없어서 Remotion에서 애니메이션으로 대체해도 되나?"
   - 예상 행동: 가능하나, 아키텍처 애니메이션(다이어그램)만 제시하고, 실제 제품 화면은 정적 스크린샷/목업으로 구성
   - 근거: 스크린 레코딩이 없으면 신뢰도 감소 → 시각적 진정성(authenticity)를 위해 실제 화면 캡처 강권

4. **QA 진행 중 오디오-비주얼 싱크 오차 감지**
   - 입력: "나레이션 음성과 자막이 0.5초 밀려있어요."
   - 예상 행동: 오프셋 값 계산 → `<Sequence from={offset_in_frames}>` 재조정 → 1회 재렌더 → QA 재점검
   - 근거: Remotion은 프레임 기반 타이밍 제어 → 간단한 오프셋 수정은 전체 재렌더링 대비 빠름

5. **총 영상 길이와 콘텐츠 밀도 불균형**
   - 입력: "Hook(5초)인데 메시지가 너무 많아서 읽을 수 없어요. 어떻게 해요?"
   - 예상 행동: Hook 텍스트 재압축 (1-2 문장으로) 또는 Hook 시간 연장(5초→10초)으로 해결
   - 근거: 텍스트 노출 시간 = (글자 수 ÷ 12) + 1초 (아이트래킹 기준) → 시간 확보 없으면 가독성 악화
