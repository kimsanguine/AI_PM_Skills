# Domain Context — pptx-ai-slide 스킬

## 1) Domain Scope

**이 스킬이 소유하는 영역:**
- PPTX 생성 및 편집 (에이전트 프로젝트 시각화)
- 슬라이드 구조 설계 (피치 덱, 아키텍처 설명, 이해관계자 업데이트)
- 청중별 메시지 밀도 최적화

**이 스킬이 소유하지 않는 영역:**
- 에이전트 설계 자체
- 슬라이드 내용의 기술적 검증

## 2) Primary Users

- **PM/리더**: 피치 덱 및 이해관계자 발표
- **마케팅**: 제품 소개 발표
- **개발팀**: 기술 설명 프레젠테이션

## 3) Required Inputs

**필수 입력:**
1. 발표 주제 (에이전트명)
2. 청중 유형 (C-Level/PM/엔지니어/투자자)
3. 핵심 메시지 3개 이내
4. 권장 슬라이드 수

**선택 입력:**
- 기존 슬라이드 또는 템플릿
- 데이터/차트

## 4) Output Contract

**산출물:**
- 완성된 PPTX 파일
- 1슬라이드 1메시지 원칙 준수
- 텍스트 overflow 0건
- 아키텍처/HITL 다이어그램 포함

## 5) Guardrails

**라우팅 규칙:**
- 슬라이드 영상 변환 → `agent-demo-video`
- 에이전트 아키텍처 설계 → `atlas/orchestration`
- 이해관계자 전략 → `stakeholder-map`

**품질 기준:**
- 1슬라이드 1메시지
- 청중별 슬라이드 수 범위 내
- 텍스트 overflow 0건

## 6) Working Facts

**청중별 슬라이드 권장 수:**

| 청중 | 슬라이드 수 | 메시지 밀도 | 포커스 |
|------|----------|----------|--------|
| C-Level | 5~8 | 낮음 | ROI, 비즈니스 임팩트 |
| PM/기획 | 8~12 | 중간 | 워크플로우, 로드맵 |
| 엔지니어 | 10~15 | 높음 | 아키텍처, 기술 스택 |
| 투자자 | 10~12 | 중간 | 시장, 차별화, 비전 |

**에이전트 피치 덱 구조:**

| 슬라이드 | 내용 | 핵심 |
|---------|------|------|
| 1 | 문제 정의 | 현재 비효율 |
| 2 | 솔루션 | 에이전트의 역할 |
| 3 | 아키텍처 | 어떻게 동작하는가 |
| 4 | 데모/워크플로우 | 실제 모습 |
| 5 | HITL 설계 | 사람은 어디에? |
| 6 | 비용/ROI | 얼마나 절감? |
| 7 | 리스크 & 완화 | 잘못되면? |
