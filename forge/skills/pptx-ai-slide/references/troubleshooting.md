# Troubleshooting — PPTX AI Slide

## 1) 청중/목적 불명확하면 슬라이드 구조 결정 불가

- **증상**: "발표자료 만들어줄 수 있을까?"만 있고, 청중/목적/분량 불명확
- **확인**:
  - 청중 유형이 명시되었는가? (C-Level / PM / Engineering / Investor)
  - 발표 목적이 명시되었는가? (이해도? 승인? 피칭? 데모?)
  - 권장 슬라이드 수 범위를 확인할 수 있는가?
- **조치**:
  1. Step 1 "요구 정렬"에서 최소 3개 필수 질문 (청중/목적/분량)
  2. 각 선택지별 권장 슬라이드 수 테이블 제시:
     ```
     C-Level: 5-8장 (메시지 밀도 낮음)
     PM/기획팀: 8-12장 (메시지 밀도 중간)
     엔지니어링: 10-15장 (메시지 밀도 높음)
     ```
  3. 사용자가 명확히 선택할 때까지 Step 2로 진행 안 함

---

## 2) 텍스트 Overflow (슬라이드 범위를 벗어남)

- **증상**: 슬라이드에 텍스트/다이어그램이 박스를 벗어나 렌더링됨
- **확인**:
  - 각 슬라이드의 텍스트가 기본 레이아웃 박스 내에 들어가는가?
  - 다이어그램 크기가 슬라이드 비율에 맞는가?
- **조치**:
  1. "1슬라이드 1메시지 원칙 재적용" → 내용 분할
  2. 슬라이드 분할 제안:
     ```
     현재: "아키텍처 + 데이터 흐름 + 에러 처리" (1장)
     분할: "아키텍처" (1장) + "데이터 흐름" (1장) + "에러 처리" (1장)
     ```
  3. 각 슬라이드의 bullet point 최대 5개로 제한

---

## 3) 아키텍처/워크플로우 다이어그램 누락

- **증상**: 기술 슬라이드에 다이어그램 없이 텍스트만 존재 (예: "API → Agent → DB")
- **확인**: 아키텍처 슬라이드에 Mermaid 또는 SVG 다이어그램이 있는가?
- **조치**:
  1. "시스템 설명은 다이어그램으로만 가능합니다"로 명시
  2. Mermaid 다이어그램 자동 제안:
     ```mermaid
     flowchart LR
         User["사용자"] --> API["API"]
         API --> Agent["에이전트"]
         Agent --> DB["데이터베이스"]
         DB --> Agent
         Agent --> User
     ```
  3. 복잡한 경우 여러 다이어그램으로 분할
     - Orchestration: flowchart
     - Data Flow: sequenceDiagram
     - State: stateDiagram-v2

## 4) HITL (Human-in-the-Loop) 슬라이드 누락

- **증상**: 슬라이드 완성 후 QA 점검 시 "사람은 어디에 개입하는가?" 설명이 없음
- **확인**: 최종 슬라이드 목록에 HITL 섹션(1장 이상)이 있는가?
- **조치**:
  1. "에이전트 발표에서 청중이 가장 먼저 묻는 질문: '결국 사람이 어디에 나와?'" 설명
  2. HITL 슬라이드 추가 (최소 1장):
     - 에이전트 출력 후 → 사람 검토
     - 사람 승인/거부 → 액션
     - 불확실한 경우 → 에스컬레이션
  3. 다이어그램으로 시각화 추천 (flowchart with decision diamonds)

## 5) 청중별 메시지 밀도 부적절 (C-Level에게 너무 자세한 기술 정보)

- **증상**: C-Level 대상 발표인데 "AWS VPC 설정", "Python 라이브러리" 등 기술 깊이가 높음
- **확인**: 청중별 권장 메시지 밀도를 따르고 있는가?
- **조치**:
  1. 청중별 메시지 밀도 기준 재확인:
     ```
     C-Level: 1 메시지/장 (비즈니스 임팩트, ROI, 리스크만)
     PM/기획: 2-3 메시지/장 (워크플로우, HITL, 로드맵)
     엔지니어링: 4-5 메시지/장 (아키텍처, 기술 스택, 배포, 모니터링)
     ```
  2. C-Level 슬라이드 재구성:
     ```
     ❌ 기술 정보: "AWS VPC, Lambda, RDS 사용"
     ✅ 비즈니스: "$500/월 비용 절감, 의사결정 속도 50% 향상"
     ```

## 6) 1슬라이드 1메시지 원칙 위배 (너무 많은 정보)

- **증상**: 한 슬라이드에 "시스템 아키텍처 + 데이터 흐름 + 에러 처리" 3가지 정보
- **확인**: 각 슬라이드가 명확한 "제목 1줄 + 핵심 메시지 1개 + 지원 비주얼"로 구성되었는가?
- **조치**:
  1. "1슬라이드 = 1메시지" 원칙 재적용
  2. 복합 정보는 슬라이드 분할:
     ```
     슬라이드 3: "시스템 아키텍처"
     슬라이드 4: "데이터 흐름"
     슬라이드 5: "에러 처리 및 복구"
     ```
  3. 각 슬라이드의 본문 bullet 최대 5개로 제한

---

## 7) 외부 소스 인용 또는 저작권 표시 누락

- **증상**: 다이어그램이나 차트가 외부 출처인데 출처 명시 안 됨
- **확인**: 모든 외부 콘텐츠에 출처가 기록되었는가?
- **조치**:
  1. Step 3 "콘텐츠 매핑" 단계에서 외부 소스 표시
  2. 최종 슬라이드 또는 부록에 "출처" 섹션 추가:
     ```
     - 시스템 아키텍처 다이어그램: [Internal Design]
     - ROI 차트: [Finance Dept. 2026-03]
     - 벤치마크 데이터: [Industry Report, Source URL]
     ```
