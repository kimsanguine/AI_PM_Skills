# Domain Context — claude-md 스킬

## 1) Domain Scope

**이 스킬이 소유하는 영역:**
- 프로젝트 디렉토리 자동 스캔 및 기술 스택 감지
- CLAUDE.md 생성/개선 (프로젝트 레벨 컨텍스트 문서)
- AI_PM_Skills 플러그인/스킬 추천 (프로젝트 시그널 기반)

**이 스킬이 소유하지 않는 영역:**
- 에이전트 시스템 프롬프트 설계 (→ `forge/instruction`)
- 프롬프트 최적화 (→ `forge/prompt`)
- 컨텍스트 윈도우 토큰 예산 (→ `forge/ctx-budget`)
- PRD 문서화 (→ `forge/prd`)
- 프로젝트 빌드/배포 실행 (범위 밖)

## 2) Primary Users

- **Claude Code 처음 쓰는 사람**: CLAUDE.md가 뭔지 모르거나, 직접 쓰기 어려운 사용자
- **기존 프로젝트에 Claude Code 도입하는 팀**: 프로젝트 맥락을 빠르게 문서화하고 싶은 개발자/PM
- **AI_PM_Skills 입문자**: 35개 스킬 중 어디서 시작해야 할지 모르는 사용자
- **CLAUDE.md가 오래된 프로젝트**: 기술 스택이 바뀌었는데 CLAUDE.md가 안 따라간 경우

## 3) Required Inputs

**필수 입력:**
1. 프로젝트 경로 (디렉토리 접근 가능)

**선택 입력:**
- 프로젝트 설명 (자동 감지가 어려운 맥락 보충)
- 기존 CLAUDE.md (개선 모드)
- 중점 영역 지정 ("에이전트 중심으로", "프론트엔드 중심으로" 등)

## 4) Output Contract

**산출물:**

| 산출물 | 보증 |
|--------|------|
| CLAUDE.md 파일 | 프로젝트 루트에 저장, 1,500~3,000 tokens |
| 기술 스택 요약 | 자동 감지 결과 (정확도 표시) |
| 빌드/테스트 명령어 | 실제 동작 확인된 것만 포함 |
| 코드 컨벤션 | 프로젝트 실제 코드에서 추론 |
| 아키텍처 설명 | 디렉토리 구조 기반 |
| 주의사항 (Anti-Goals) | 최소 3개 |
| AI_PM_Skills 추천 | 프로젝트 시그널 기반, 관련 있는 것만 |

## 5) Guardrails

**라우팅 규칙:**
- CLAUDE.md 생성 완료 후 에이전트 프로젝트 → `forge/instruction` 제안
- CLAUDE.md 생성 완료 후 프롬프트 개선 필요 → `forge/prompt` 제안
- 토큰 예산 최적화 필요 → `forge/ctx-budget` 제안

**품질 기준:**
- 프로젝트 스캔 없이 CLAUDE.md를 생성하지 않음 (일반 템플릿만 제공 금지)
- 모든 스킬을 무차별 추천하지 않음 (시그널 기반만)
- 기존 CLAUDE.md가 있으면 덮어쓰기 전 확인

## 6) Working Facts

**CLAUDE.md 섹션별 예상 토큰:**

| 섹션 | 토큰 | 우선순위 |
|------|------|---------|
| 프로젝트 개요 | 50~100 | 필수 |
| 기술 스택 | 100~200 | 필수 |
| 빌드 & 실행 | 150~300 | 필수 |
| 코드 컨벤션 | 200~400 | 필수 |
| 아키텍처 | 300~500 | 필수 |
| 주의사항 | 100~200 | 필수 |
| 에이전트 구조 | 200~400 | 조건부 (에이전트 있을 때만) |
| **Total** | **~1,100~2,100** | |

**프로젝트 유형별 CLAUDE.md 복잡도:**

| 유형 | 스캔 시간 | 섹션 수 | 추천 스킬 수 |
|------|----------|--------|------------|
| 단일 패키지 (React/Next.js) | ~30초 | 6 | 0~2 |
| Python 백엔드 (FastAPI/Django) | ~30초 | 6 | 0~2 |
| 에이전트 프로젝트 | ~1분 | 7 (에이전트 섹션 추가) | 3~5 |
| 모노레포 | ~2분 | 루트 + 패키지별 | 2~4 |

**forge 스킬 체인 내 위치:**

```
forge/claude-md   ← 시작점 (프로젝트 온보딩)
    ↓
forge/instruction  (에이전트 설계)
    ↓
forge/prompt       (프롬프트 최적화)
    ↓
forge/prd          (PRD 문서화)
```

## 7) Fill-in Checklist

- [ ] 프로젝트 디렉토리에 접근할 수 있는가?
- [ ] 기술 스택 감지 파일(package.json 등)이 존재하는가?
- [ ] 기존 CLAUDE.md 존재 여부를 확인했는가?
- [ ] 에이전트 관련 디렉토리(agents/, instructions/)를 확인했는가?
- [ ] 모노레포 여부(packages/, apps/)를 확인했는가?
- [ ] 빌드/테스트 명령어가 실제 동작하는지 검증했는가?
- [ ] 생성된 CLAUDE.md 토큰이 1,500~3,000 범위인가?
- [ ] AI_PM_Skills 추천이 프로젝트 시그널과 매칭되는가?
