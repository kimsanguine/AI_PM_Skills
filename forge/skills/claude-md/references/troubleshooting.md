# Troubleshooting — claude-md 스킬

## 1) 기술 스택 자동 감지 실패

**증상:**
- 프로젝트 디렉토리에 package.json, pyproject.toml 등 설정 파일이 없음
- 스캔 결과가 빈 상태
- "설정 파일 없음"으로 표시됨

**확인:**
- 디렉토리 경로가 정확한가? (`pwd`로 확인)
- 설정 파일이 서브디렉토리에 있는가? (모노레포 구조)
- 프로젝트가 아직 초기화 전인가?

**조치:**
- 서브디렉토리 스캔 깊이 확대 (`-maxdepth 3`)
- 설정 파일 없으면 수동 입력 모드 전환
- 사용자에게 "어떤 기술 스택을 사용하나요?" 질문

---

## 2) 빌드 명령어가 실제 동작하지 않음

**증상:**
- CLAUDE.md에 `npm run build`가 기록됐지만 실행 시 에러
- package.json scripts에 해당 명령어가 없음

**확인:**
- `cat package.json | jq '.scripts'`로 실제 스크립트 확인
- `npm run` (인자 없이)으로 사용 가능한 스크립트 목록 확인

**조치:**
- 실제 존재하는 스크립트만 기록
- 존재하지 않는 명령어는 "설정 필요: build 스크립트가 정의되지 않음"으로 표시
- `Makefile`이 있으면 make 타겟도 확인

## 3) 기존 CLAUDE.md와의 충돌

**증상:**
- 이미 CLAUDE.md가 존재하는데 새로 생성 시도
- 기존 내용이 사라질 위험

**확인:**
- `cat CLAUDE.md`로 기존 내용 확인
- 기존 내용의 품질 평가 (섹션 수, 구체성, 토큰 수)

**조치:**
- 항상 개선 모드 우선 제안
- 새로 생성 시 기존 파일을 `.claude-md.backup`으로 백업
- diff 형태로 변경점 설명 후 사용자 확인

---

## 4) 모노레포에서 CLAUDE.md 범위 혼란

**증상:**
- 모노레포(packages/, apps/)인데 루트에만 CLAUDE.md 생성
- 각 패키지의 특화 정보가 누락

**확인:**
- `ls packages/ apps/ 2>/dev/null`로 모노레포 구조 확인
- 각 패키지에 독립 package.json이 있는가?

**조치:**
- 루트 CLAUDE.md: 전체 구조, 공통 명령어, 모노레포 설명
- 패키지별 CLAUDE.md: 해당 패키지 특화 정보만
- "이 프로젝트는 모노레포 구조입니다. 루트 + 패키지별 CLAUDE.md를 생성할까요?" 선택지 제공

## 5) AI_PM_Skills 추천이 과도하거나 부적절

**증상:**
- 에이전트 코드가 없는데 에이전트 스킬 추천
- 모든 5개 플러그인을 무조건 추천

**확인:**
- agents/, instructions/ 디렉토리 존재 여부
- 에이전트 관련 키워드가 코드에 있는지 (`grep -r "agent" --include="*.ts"`)
- PRD, 경쟁분석 등 PM 문서 존재 여부

**조치:**
- 프로젝트 시그널 → 추천 매핑 테이블을 엄격히 적용
- 시그널이 없는 플러그인은 추천하지 않음
- "에이전트 관련 코드가 감지되지 않아 에이전트 스킬 추천을 건너뜁니다" 명시

## 6) CLAUDE.md 토큰이 너무 길어짐

**증상:**
- 생성된 CLAUDE.md가 5,000+ tokens
- 컨텍스트 윈도우 낭비

**확인:**
- 각 섹션별 토큰 수 확인
- 불필요하게 상세한 섹션이 있는지 확인

**조치:**
- 목표: 1,500~3,000 tokens
- 상세 정보는 별도 문서로 분리하고 CLAUDE.md에서 링크
- 예: 상세 아키텍처 → `docs/architecture.md`, 상세 컨벤션 → `.eslintrc` 직접 참조
- "CLAUDE.md가 3,000 tokens를 초과합니다. 다음 섹션을 별도 파일로 분리할까요?" 제안
