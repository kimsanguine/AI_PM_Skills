# Bad Example — agent-plan-review 스킬

## 사용자 요청

"에이전트 검토해줄래?"

## 거절 이유

- 어떤 에이전트인지 불명확
- 검토할 문서(PRD/아키텍처/instruction)가 제시되지 않음
- 검토 깊이(축소/전체/압축) 미정의

## 올바른 라우팅

1. 에이전트 이름/설계 문서 제공
2. 검토 범위 명시 (축소/전체/압축)
3. 스킬 호출

## 수정 방향

"morning-briefing 에이전트 설계를 구현 전에 완전히 검토해줄래? 설계 문서: PRD(7섹션 완료), 아키텍처 다이어그램, Instruction 정의돼있어. 범위: 전체 리뷰로 아키텍처, 인스트럭션, 신뢰성, 비용까지 모두 봐줄래?"
