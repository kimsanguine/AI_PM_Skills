# Good Example — agent-plan-review 스킬

## 사용자 요청

"morning-briefing 에이전트 설계를 구현 전에 완전히 검토해줄래? PRD가 있고, 아키텍처 다이어그램도 있는데, 구현하면서 놓칠 수 있는 게 있을까봐."

## 승인 이유

- 구현 전 에이전트 설계를 4축(범위, 아키텍처, 인스트럭션, 신뢰성)으로 검증 필요
- 비용 낭비와 재작업 방지
- 기존 자산 재사용 기회, 복잡도 경고, 장애 모드 사전 식별

## 예상 처리

1. Step 0 범위 점검: 기존 자산 재사용 가능성 / MVA 확인 / 복잡도 경고
2. 아키텍처 리뷰: 오케스트레이션 패턴 / 에이전트 간 경계 / 데이터 흐름 / 모델 선택 / 메모리 설계 / 단일 장애 지점
3. 인스트럭션 품질 리뷰: 7요소 완성도 / 프롬프트 효율 / 컨텍스트 예산 / 가드레일
4. 운영 신뢰성 리뷰: 환각 대응 / 컨텍스트 드리프트 / 비용 폭등 / 에스컬레이션 경로 / 모니터링 훅 / 롤백 전략
5. 비용 & 스케일 리뷰: 토큰 단가 추정 / 스케일 시나리오 / 캐싱 기회 / 모델 다운그레이드

## 최종 결과물

구현 전 설계의 이슈를 명확히 하고, 각 이슈에 대해 의견이 담긴 권고안과 트레이드오프를 제시하는 리뷰 보고서
