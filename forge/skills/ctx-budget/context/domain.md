# Domain Context — ctx-budget 스킬

## 1) Domain Scope

**이 스킬이 소유하는 영역:**
- 에이전트 한 세션의 컨텍스트 윈도우 입출력 관리
- 파일/데이터의 토큰 소비 추정
- 필수/조건부/제외 파일 분류
- 동적 로딩 전략 설계

**이 스킬이 소유하지 않는 영역:**
- 장기 저장소 설계 (데이터베이스) (→ `atlas/memory-arch`)
- 프롬프트 토큰 최적화 (→ `forge/prompt`)

## 2) Primary Users

- **에이전트 설계자**: 메모리 전략의 토큰 예산 계획
- **구현팀**: 파일 로드 우선순위 구현
- **운영팀**: 컨텍스트 사용률 모니터링

## 3) Required Inputs

**필수 입력:**
1. 로드할 파일 목록 (이름, 크기 추정)
2. 에이전트가 사용할 모델 (Haiku/Sonnet/Opus)
3. 현재 토큰 사용률 (알면)

**선택 입력:**
- 동적 로딩 도구 (memory_search 가능 여부)

## 4) Output Contract

**산출물:**
- 필수/조건부/제외 파일 분류
- 각 파일별 토큰 추정값
- 70%/85% 임계값별 모니터링 계획
- 동적 로딩 조건 정의

| 항목 | 보증 |
|------|------|
| 파일 분류 | 필수/조건부/제외 구분 |
| 토큰 추정 | 각 파일별 + 총합 |
| 동적 로딩 | 로딩 조건 명시 |
| 임계값 | 70%/85% 모니터링 계획 |

## 5) Guardrails

**라우팅 규칙:**
- 프롬프트 토큰 최적화 → `forge/prompt`
- 메모리 아키텍처 전체 설계 → `atlas/memory-arch`

**품질 기준:**
- 70% / 85% 임계값은 경험 기반 가이드 (절대값 X)
- 토큰 추정은 모델별로 다름 (Haiku/Sonnet/Opus 명시)

## 6) Working Facts

**모델별 실용적 한도:**

| 모델 | 최대 컨텍스트 | 실용 한도 | 권장 목표 |
|------|------------|---------|---------|
| Haiku | 200k | 40k | <28k (70%) |
| Sonnet | 200k | 80k | <56k (70%) |
| Opus | 200k | 100k | <70k (70%) |

**토큰 환산율:**
- 마크다운: 1KB ≈ 250~350 tokens
- 코드: 1KB ≈ 200~300 tokens
- JSON: 1KB ≈ 150~250 tokens
- 일반 텍스트: ~750 단어 = 1,000 tokens

**파일 로딩 기준:**

| 파일 유형 | 파일 크기 | 토큰 추정 | 분류 |
|----------|---------|---------|------|
| SOUL.md | ~2KB | ~600 | 필수 |
| USER.md | ~3KB | ~900 | 필수 |
| MEMORY.md | ~10KB | ~3,000 | 조건부 |
| 일별 메모리 | ~5KB | ~1,500 | 필수/조건부 |
| SKILL.md | ~4KB | ~1,200 | 조건부 |

**TO BE UPDATED by reviewer:**

- 조직별 응답 품질 기준 (70% vs 60%?)
- memory_search 도구 가용성
