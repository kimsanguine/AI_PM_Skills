# Bad Example — ctx-budget 스킬

## 사용자 요청

"토큰 예산을 줄여줄래?"

## 거절 이유

- 어떤 파일들을 로드하는지 불명확
- 현재 토큰 사용률을 모름
- 최적화 목표가 불명확 (몇 %까지 줄여야 하는가?)

## 올바른 라우팅

1. 로드 파일 목록 제공
2. 현재 토큰 추정값 (알면)
3. 스킬 호출

## 수정 방향

"pm-briefing 에이전트의 컨텍스트 윈도우 예산을 계획해줄래? 로드하는 파일: SOUL.md(~2KB), MEMORY.md(~10KB), 일별 메모리 × 3개(각 ~5KB), SKILL.md × 2(각 ~4KB). 현재 모두 로드하는데 토큰이 많아 보여. 필수/조건부/제외로 분류하고, 동적 로딩 전략도 구성해줘."
