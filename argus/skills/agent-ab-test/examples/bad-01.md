# Bad Example — Agent A/B Test

## 사용자 요청

"새 프롬프트가 더 좋아 보이는데, 우리 팀에서 써봤을 때 확실히 B가 나았어. 배포해도 되겠지?"

## 거절 이유

- 가설 없이 주관적 인상으로 결정
- 통계적 검증 없음 (팀 주관적 평가만)
- 표본 크기 미계산 (몇 명이 테스트했는가?)
- p-value/신뢰 구간 없음 (실제 개선인지 우연인지 구분 불가)
- Guardrail 메트릭 미설정 (비용이나 정확도 악화는 무시?)
- A/B 동시 실행 미확인 (시간대 편향 가능)
- 3건 중 2건이 B에서 성공하는 것을 "B가 더 좋다"로 착각 가능

## 올바른 라우팅

**argus/skills/agent-ab-test** (통계적 설계 및 검증 필요)

## 수정 방향

"새 프롬프트로 A/B 테스트를 할 수 있어? 필요 표본 크기를 계산하고 (MDE 5% 기준), A/B를 동시에 실행해서 시간대 편향을 통제해줘. 정확도(Primary), 응답 시간(Secondary), Hallucination(Guardrail)을 모니터링하고 p-value > 0.05인지 확인해줘. 그리고 의사결정(Ship/Keep/Iterate)을 통계 기반으로 내려줄 수 있어?"
