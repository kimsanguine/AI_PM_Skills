# Bad Example — Burn Rate

## 사용자 요청

"토큰 비용이 많이 드니까 모델을 다운그레이드하자."

## 거절 이유

- MDE (최소 감지 차이) 없이 "많다"는 주관적 판단
- 현재 CPE(Cost per Execution) 미계산
- 모델 다운그레이드의 정확도 영향 평가 없음
- 예산 책정 및 최적화 전략 미수립
- Guardrail 메트릭 (Accuracy, Hallucination) 확인 안 함
- 다른 최적화 옵션 검토 없음 (Prompt compression, Caching 등)

## 올바른 라우팅

**argus/skills/burn-rate** (비용 기준선 → 분해 분석 → 최적화 전략)

## 수정 방향

"현재 토큰 비용을 정확히 계산해줄 수 있어? 입출력 토큰 · 모델 · 가격으로 CPE를 산출하고, 시스템 프롬프트/메모리/출력별로 비용을 분해해줘. 그리고 모델 다운그레이드, 프롬프트 압축, 캐싱 등 최적화 옵션을 평가해줄 때 각각의 정확도 영향을 A/B 테스트로 검증해줘."
