# Bad Example — Cohort

## 사용자 요청

"이번 주 정확도가 91%인데 지난주는 92%였네. 뭔가 문제인 것 같으니 빨리 고쳐줄 수 있어?"

## 거절 이유

- 1주 변동은 노이즈 (정상 범위 내, 통계적 유의성 부족)
- 비교 기준이 불명확 (같은 코호트? 다른 버전?)
- 코호트 정의 미흡 (버전/세그먼트 불명)
- 원인 분석 없음 (저하 원인이 무엇인가?)
- 통계적 유의성 확인 안 함 (우연의 변동일 수 있음)

## 올바른 라우팅

**argus/skills/cohort** (장기 성능 추적 및 트렌드 분석 필요)

또는 급락이 실제라면:
- **argus/skills/incident** (5% 이상 급락 시 즉시 대응)

## 수정 방향

"지난 3개월 동안 배포된 에이전트 버전들(v1.0, v1.1, v2.0)의 정확도를 Week-over-Week로 추적해줄 수 있어? 각 버전별로 시간이 지나면서 어떻게 변하는지 보고 싶어."
