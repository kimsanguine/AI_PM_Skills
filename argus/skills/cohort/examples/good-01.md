# Good Example — Cohort

## 사용자 요청

"우리가 3개월 전에 새로운 고객 지원 에이전트를 배포했는데, 정확도가 처음에는 93%였다가 지금은 88%로 떨어졌어. 버전별로 성능 추이를 추적해서 어디가 문제인지 파악하고, 개선 방안을 제시해줄 수 있어?"

## 승인 이유

- 버전별(시간 기반) 코호트 분석 명시
- 성능 저하 추세 추적 요청 (Week-over-Week 비교 암묵)
- 원인 분석과 액션 아이템 연결

## 예상 처리

1. 배포 이력 확인 (v1.0, v1.1, v2.0 등)
2. 각 버전별 첫 1주, 4주, 12주 정확도 추적
3. Week-over-week 비교로 저하 패턴 식별
4. 원인 분석 (데이터 드리프트? 프롬프트 노후화? 사용 패턴 변화?)
5. 성능 저하가 버전 간 차이인지 시간 경과 효과인지 판단
6. 개선 액션 (프롬프트 리프레시, 모델 업그레이드, 데이터 보강)
