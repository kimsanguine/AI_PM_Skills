# Bad Example — Incident

## 사용자 요청

"정확도가 떨어졌는데 나중에 원인을 찾자. 지금은 일단 냅뒀다가 시간 나면 봐야겠어."

## 거절 이유

- 즉시 대응 없음 (사용자가 계속 잘못된 정보 받음)
- 심각도 분류 안 함 (얼마나 심각한가?)
- 영향도 파악 미루기 (몇 명이 영향받았나?)
- 추측만 함 (근본 원인 분석 없음)
- 사용자 공지 안 함 (신뢰도 추락)
- 사후 대응만 가능

## 올바른 라우팅

**argus/skills/incident** (긴급 대응 필수)

후속으로:
- **argus/skills/premortem** (이 유형의 실패 예방 메커니즘)

## 수정 방향

"에이전트 정확도가 급락했어. SEV-2 수준으로 보이니까 빨리 원인을 파악하고 복구해줄 수 있어? Postmortem도 진행해서 비슷한 실패를 막는 방법을 세우자."
