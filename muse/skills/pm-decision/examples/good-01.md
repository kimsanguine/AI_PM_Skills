# Good Example — pm-decision 패턴 매칭

## 사용자 요청
"우리가 지금 스코프를 줄일지 일정을 늘릴지 결정해야 하는데,
예전에 비슷한 상황에서 어떤 패턴으로 판단했는지 매칭해줘.
인지 편향 체크도 함께 해줄래."

## 승인 이유
- pm-decision의 핵심: 반복되는 의사결정 패턴을 라이브러리에서 찾아 현재 상황에 적용
- 스코프 vs 일정 트레이드오프는 PM이 매일 맞닥뜨리는 반복 의사결정
- 패턴 라이브러리와 "함정 체크" 기능이 정확히 필요한 상황

## 예상 처리
1. 상황 분석: "현재 진행 중인 에이전트 개발, MVP 기능 7개 중 4개 완료, 6주 마감"
2. 패턴 매칭: "Scope Creep Prevention" 패턴 검색 → 도움 되지만, "Scope vs Timeline" 직접 패턴이 더 적절
3. 가장 유사 패턴 제시: "Why-First Decision Making" + "Scope Creep Prevention" 조합
4. 판단 기준 적용:
   - Why-First: "6주 마감이 왜 중요한가? (고객 commitment vs 내부 목표?)"
   - 함정 체크: "스코프를 늘리려는 이유가 '좋은 아이디어'라는 감정은 아닌가?"
5. 최종 권장: "마감을 우선 지키되, MVP 범위를 명확히 재정의하고 나머지는 Phase 2로 백로그"
6. 인지 편향 체크: "Sunk Cost Fallacy 주의 (이미 4개 완료했으니 나머지도 넣자 X), Planning Fallacy 주의 (2주면 끝날 줄 아는 것)"
