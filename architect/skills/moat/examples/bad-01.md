# 나쁜 예제 — Agent Moat

## 사용자 요청
"우리 에이전트는 정말 좋은 UI를 가지고 있고, 우리가 먼저 시장에 나왔으니까, 경쟁사가 따라오기 어려울 것 같아."

## 거절 이유
- **UI는 Moat이 아님**: 3개월이면 경쟁사도 복제 가능
- **First-mover는 약한 우위**: 특히 AI 에이전트 시장에서 (모델 성능 개선 빠름)
- **모두 False Moat**: 기술이 commodity화되면 무의미
- 실제 방어 가능한 moat (데이터, 워크플로우, 네트워크) 미검토

## 올바른 라우팅
- **6가지 Moat 유형을 모두 평가**: Data Flywheel, Workflow Lock-in, Network Effects 등
- **실제 경쟁 우위 식별**: 경쟁사가 따라 오기 어려운 것이 무엇인지
- **구축 로드맵 수립**: Phase 1-4로 시간별로 moat 강화 계획

## 수정 방향
"우리 제품의 진정한 경쟁 우위가 무엇일까?" → Moat 유형 평가로 6가지 재검토
"고객 데이터가 우위가 될 수 있나?" → Growth-loop와 연계해서 데이터 플라이휠 설계
"워크플로우에 깊이 있게 통합할 방법이 있을까?" → Workflow Lock-in 수준 평가
