# 좋은 예제 — Agent Moat

## 사용자 요청
"우리 법률 문서 검토 에이전트가 경쟁사와 어떤 점에서 다르게 유지될 수 있을까? 그리고 경쟁이 심해지면 우리의 방어선은 뭘까?"

## 승인 이유
- 에이전트 제품의 장기 경쟁력(moat) 평가 및 설계 필요
- 초기 우위(빠른 성장)와 장기 방어(모방 방지)의 균형 필요
- 시장 진입 장벽을 높이는 구체적 전략 필요

## 예상 처리
1. Moat 유형 평가: 6가지 유형(Data Flywheel, Workflow Lock-in, Network Effects 등) 각각 점수화
2. 주요 Moat 선정: 가장 강한 1-2개 선택 + 근거
3. 취약점 분석: 각 Moat의 가장 큰 위협 식별
4. 구축 로드맵: Phase 1-4로 시간별 투자 계획
5. False Moat 제거: "우리 UI가 더 좋다" 같은 근거 없는 선택 제거

## 처리 결과 예시
```
제품: 법률 문서 검토 에이전트

주요 Moat:
1. Data Flywheel (4/5): 고객 데이터 독점, 한국 법조문 적중률 95% vs 경쟁사 60%
2. Proprietary Knowledge (4/5): 법률 전문가 팀 필요, 6개월 소요

구축 로드맵:
- Phase 1 (0-3개월): 검토 속도 50% 단축으로 초기 고객 확보
- Phase 2 (3-6개월): 한국 법조문 전문성 강화
- Phase 3 (6-12개월): 고객 피드백으로 프롬프트 지속 개선
- Phase 4 (12+개월): 다른 법적 영역 확장

전체 Moat 강도: 2.4/5 (현재) → 4.2/5 (12개월)
```
