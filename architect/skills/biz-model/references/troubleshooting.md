# Troubleshooting — biz-model 스킬

## 1) 가격 대비 가치 명확하지 않음

**증상:**
- "고객에게 가격을 설명할 때 이해하지 못한다"
- "왜 이 가격인가요?" 질문 반복
- 전환율 <5% (낮음), 구매 거절 증가

**확인:**
- 현재 가치 주장(Value Proposition):
  - "월 시간 절감: 8시간"인가?
  - "월 비용 절감: $2,400"인가?
  - 명확한가?
- 고객 인터뷰:
  - 실제 고객이 얼마나 절감하는가?
  - 고객의 주요 통증점(Pain Point)은?

**조치:**
1. 가치 재계산:
   - 3-5명의 실제 고객 인터뷰
   - "우리 에이전트를 쓰기 전후 비교"
   - 실제 시간 절감, 비용 절감, 새로운 기능 가능 범위 측정
2. 가치 전 재작성:
   ```
   나쁜 예: "우리 에이전트는 스마트합니다"
   좋은 예: "상담사 1명이 월 64시간 절감, 월급 960,000원 절감"
   ```
3. 가격 재설정:
   - 절감액의 10-20% 청구 기준
   - 절감액 $960,000 × 10% = $96,000/년 가격
4. 고객 세그먼트별 가치 분석:
   - 스타트업 (시간 절감 중심)
   - 대기업 (비용 절감 중심)
   - 각각에 맞는 가치 전 작성

## 2) CPE(Cost Per Execution)가 이윤율 목표를 초과

**증상:**
- "CPE가 $0.08, 가격이 $0.10인데 마진이 20%밖에 안 된다"
- "인프라, 지원 비용 합치면 음수다"
- Gross Margin < 50% (목표 미달)

**확인:**
- CPE 구성 요소:
  - LLM API: $0.05
  - 인프라: $0.02
  - 지원: $0.01
  - 합: $0.08
- 수익: $0.10
- 마진: ($0.10 - $0.08) / $0.10 = 20%

**조치:**
1. 원가 최적화 (시급):
   - **LLM 비용 최적화**:
     - 모델 라우팅 (Sonnet → Haiku 75% 작업)
     - 프롬프트 캐싱 (20-30% 절감)
     - 배치 처리 (10% 절감)
     - 목표: $0.05 → $0.03 (-40%)
   - **인프라 비용**:
     - 서버 스케일 최적화
     - 자동 스케일 다운
     - 목표: $0.02 → $0.01 (-50%)
   - **지원 비용**:
     - 자동화 (FAQ 챗봇)
     - 목표: $0.01 → $0.005 (-50%)
2. 수정 후 CPE:
   - 기존: $0.08
   - 최적화 후: $0.035 (-56%)
   - 새 마진: ($0.10 - $0.035) / $0.10 = 65% ✓
3. 가격 인상 (원가 최적화만으로 부족):
   ```
   기존: $0.10
   시장 분석: 경쟁사 $0.15
   고객 가치: $1.00 (절감액 대비)
   재설정: $0.15 (경쟁사 수준)
   새 마진: ($0.15 - $0.035) / $0.15 = 77%
   ```
4. 고객 커뮤니케이션:
   - "원가 절감 효과를 고객과 공유" (가격 유지 또는 소폭 인상)
   - "가격 인상 = 개선된 품질" 메시지

## 3) 가격 모델과 고객 사용 패턴 불일치

**증상:**
- "사용 기반 가격인데 고객마다 사용량이 10배 차이가 난다"
- "어떤 고객은 월 100건, 어떤 고객은 10,000건"
- "고객 수익도 다양하고, 비용도 예측 불가능"

**확인:**
- 고객 사용 분포:
  - Median: 500건/월
  - P25: 100건/월, P75: 2,000건/월
  - 범위: 10배 차이
- 가격 모델:
  - 사용 기반: $0.10/건
- 고객별 월 비용:
  - 저사용: $10
  - 중사용: $50
  - 고사용: $200

**조치:**
1. 하이브리드 가격 모델로 전환:
   ```
   기본: $50/월 (최대 500건 포함)
   초과분: $0.10/건 (500건 이상)

   효과:
   - 저사용: $50 (고정)
   - 중사용: $50 + $150 = $200
   - 고사용: $50 + $1,700 = $1,750
   ```
2. 티어드 구독 모델:
   ```
   Starter: $50/월 (최대 500건)
   Pro: $200/월 (최대 5,000건)
   Enterprise: Custom (10,000건+)

   장점: 고객 수익 예측 가능, 회사 수익도 예측 가능
   ```
3. 시간당 구독 (팀 도구용):
   ```
   팀 3명: $300/월
   팀 10명: $800/월

   장점: 사용량과 무관, 고정 비용 → 회사 캐시 플로우 개선
   ```
4. 고객 분류:
   - 저사용 고객: 기본 가격 ($50)로 만족 → Tier 1 유지
   - 고사용 고객: Enterprise 상담 → 높은 가격

## 4) 경쟁사 가격 급락

**증상:**
- "새로운 경쟁사가 우리 가격의 50% ($0.10 → $0.05)로 출시"
- "고객들이 경쟁사로 옮길까?"
- "우리는 가격을 내릴 수 없다. 비용이 감당이 안 된다"

**확인:**
- 경쟁사의 가격 책정 전략:
  - 진정 저원가인가?
  - 아니면 시장 침투 전략(시간 제한)인가?
- 우리의 차별화:
  - 정확도? (95% vs 경쟁사 85%?)
  - 속도? (1s vs 경쟁사 3s?)
  - 고객 지원? (24/7 vs 경쟁사 업무시간만?)

**조치:**
1. 진정한 차별화 분석:
   - "우리가 경쟁사보다 뭐가 나은가?" (정량화)
   - "고객이 그 차이를 인식하나?"
2. 원가 구조 재최적화:
   - "우리의 CPE를 $0.05 이하로 낮출 수 있나?"
   - 모델 라우팅, 캐싱, 배치 처리 극대화
   - 현실적 한계 파악
3. 가치 차별화 강화:
   - 정확도 우위: "우리는 정확도 95%, 경쟁사 85%"
   - 고객 성공: 24/7 지원, 온보딩, 커스터마이징
   - 신뢰도: 보안, 규정 준수(GDPR 등)
4. 가격 전략:
   ```
   옵션 A: 가격 유지, 가치 강조
   - "저가 = 낮은 품질"이 아님을 증명
   - 차별화된 상위 티어 출시 (Premium)

   옵션 B: 선택적 가격 인하
   - "기본" 티어만 경쟁사 가격 (박리다매)
   - "Pro" 티어는 프리미엄 가격 (고객 지원, 정확도)

   옵션 C: 번들링
   - 에이전트 + 고객 성공 서비스 = 종합 솔루션
   - 순수 가격 비교 어렵게 만들기
   ```
5. 진정한 해자 강화:
   - 데이터 피드백 루프 (moat 스킬)
   - 고객 잠금 (switching cost 높이기)
   - 네트워크 효과 (여러 팀 동시 도입)
