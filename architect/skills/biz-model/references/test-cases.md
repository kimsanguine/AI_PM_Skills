# Test Cases — biz-model 스킬

## 1) Trigger Tests

### Should Trigger (5)

1. "새로운 AI 에이전트 제품을 출시하는데 어떻게 가격을 책정할까?"
   - 이유: 제품 가격책정 전략 필요

2. "현재 모델에서 실제로 얼마나 수익을 낼 수 있을까?"
   - 이유: 단위 경제(Unit Economics) 분석

3. "사용 기반(per-use), 구독, 성과 기반 중 어떤 가격 모델이 최적인가?"
   - 이유: 가격 모델 평가 및 선택

4. "고객이 얼마나 절감하는지 알아야 우리가 얼마나 청구할지 알 수 있을 것 같다"
   - 이유: 가치 창출 vs 가치 포획 분석

5. "VC 피칭을 위해 비즈니스 케이스를 작성해야 한다"
   - 이유: 지속 가능한 비즈니스 모델 설계

### Should NOT Trigger (5)

1. "우리 제품이 경쟁사와 비교해서 뭐가 다를까?"
   - 올바른 라우팅: `moat` (경쟁 우위 분석)

2. "월간 토큰 비용을 추적하고 최적화하려면?"
   - 올바른 라우팅: `burn-rate` (비용 추적)

3. "제품을 출시할 때 어떤 Go-To-Market 전략을 써야 할까?"
   - 올바른 라우팅: `agent-gtm` (GTM 전략) 또는 `prd` (제품 출시)

4. "데이터 축적이 경쟁 우위가 될 수 있을까?"
   - 올바른 라우팅: `growth-loop` (그로스 루프 설계)

5. "이 에이전트 제품이 정말 시장에서 성공할 수 있을까?"
   - 올바른 라우팅: `growth-loop` (성장 가능성 검증)

## 2) Edge Cases

### 경계 사례 (4)

1. **가격 대비 가치 명확하지 않음**
   - 입력: "고객이 구매 가치를 이해하지 못한다"
   - 예상 행동: 고객 인터뷰 재실시, 실제 절감 금액 재계산, 가치 전 조정 또는 가격 인하
   - 근거: 가치가 명확하지 않으면 전환율 급락

2. **CPE(Cost Per Execution)가 이윤율을 초과**
   - 입력: "비용은 $0.08이고 가격은 $0.10인데 마진이 20%밖에 안 된다"
   - 예상 행동: API 비용 최적화, 배치 처리 도입, 또는 가격 인상 (고객 반발 가능성 검토)
   - 근거: Gross Margin <50%는 지속 불가능한 모델

3. **가격 모델이 고객 사용 패턴과 맞지 않음**
   - 입력: "사용 기반 가격인데 고객 사용량이 너무 편차가 크다"
   - 예상 행동: 하이브리드 모델 도입(구독 + 사용량), 또는 다른 가격 모델로 전환
   - 근거: 편차가 크면 고객과 회사 모두 예측 불가능

4. **경쟁사 가격 급락**
   - 입력: "새로운 경쟁자가 우리 가격의 50% 수준으로 출시했다"
   - 예상 행동: 원가 구조 재최적화, 또는 가치 차별화 강화 (moat)
   - 근거: 원가 경쟁만으로는 버티기 어려움, 차별화 필수
