# 나쁜 예제 — Business Model

## 사용자 요청
"경쟁사가 $500/월이니까 우리도 $500/월로 책정하자. 그리고 고객들이 충분히 사용할 거라고 가정하자."

## 거절 이유
- **고객 가치를 계산하지 않음**: 실제로 고객이 얼마나 절감하는지 모름
- **비용 구조 미검토**: CPE 계산 없이 가격 결정 → 초과 사용 시 손실 가능
- **가격 모델 비교 없음**: 왜 월정액이 좋은지, 사용량 기반과의 차이가 뭔지 평가 안 함
- **고객 검증 생략**: 추측 기반 가격 책정
- **Gross Margin 미검증**: 수익성 확인 안 함

## 올바른 라우팅
- **Trigger Gate**: 고객 가치를 **정량적으로 계산**하고 비용 구조를 분석한 후 가격 책정
- **3가지 가격 모델 검토**: Per-execution, Tiered subscription, Outcome-based
- **Gross Margin >70% 검증**: 최고 SaaS 표준
- **고객 인터뷰**: 최소 2명 이상의 실제/예상 고객과 타당성 확인

## 수정 방향
"우리 에이전트가 고객에게 얼마나 가치를 만드나?" → 구체적 수치로 계산
"우리의 변수 비용이 얼마나 되나?" → CPE 계산
"어떤 가격 모델이 우리 제품과 고객 특성에 맞을까?" → 3가지 이상 시나리오 비교
