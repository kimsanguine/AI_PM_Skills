# Pre-mortem 분석: 금융 고객지원 AI 에이전트 프로덕션 배포
**방법론: FMEA (Failure Mode and Effects Analysis)**
**대상 시스템**: 금융상품 문의 처리 에이전트 | **처리량**: 1,000건/일 | **배포일**: 다음 주

---

## FMEA 평가 기준

| 항목 | 척도 | 설명 |
|------|------|------|
| **심각도 (S)** | 1–10 | 1=미미, 10=치명적 |
| **발생빈도 (O)** | 1–10 | 1=극히 드묾, 10=거의 확실 |
| **감지가능성 (D)** | 1–10 | 1=즉시 감지, 10=감지 불가 |
| **RPN** | S × O × D | **> 100: 즉시 조치 필요** |

## 배포 전 필수 조치 (Go/No-Go Checklist)

### 🚫 No-Go 조건 (이 중 하나라도 미완료 시 배포 금지)

```
[ ] 금융정보 응답 RAG 검증 레이어 구축
[ ] 상품 DB 실시간 연동 또는 최신화 확인
[ ] 투자권유 감지 필터 + 차단 로직 구현
[ ] PII 로그 마스킹 적용 (계좌번호, 주민번호, 전화번호)
[ ] 금융소비자보호 설명의무 체크리스트 워크플로우
[ ] temperature=0 설정 (금융 정보 응답)
[ ] 보이스피싱/이상거래 패턴 탐지 활성화
```

### ⚠️ 조건부 배포 (1주 내 해결 약속 + 모니터링)

```
[ ] 에스컬레이션 로직 정의 및 테스트
[ ] 피크 부하 테스트 (1,500건/일 기준)
[ ] Blue-Green 배포 파이프라인 및 롤백 드릴
[ ] 감성 분류 모델 또는 키워드 기반 전환 로직
[ ] 대화 세션 외부 저장 (컨텍스트 유실 방지)
```

## 리스크 완화 아키텍처 요약

```
고객 입력
    │
    ▼
[PII 마스킹 레이어] ──────────────────────── 로그 저장 (마스킹 후)
    │
    ▼
[의도 분류기]
    ├── 민원/투자권유 감지 → 즉시 인간 상담원 전환
    └── 일반 문의 → 다음 단계
    │
    ▼
[RAG 파이프라인] ← 실시간 상품 DB (금리·약관·한도)
    │
    ▼
[LLM 응답 생성] (temperature=0, 금융정보)
    │
    ▼
[응답 검증 레이어]
    ├── 수치 정보 → DB 교차 검증
    ├── 설명의무 체크 → 필수 안내 포함 확인
    └── 사기 패턴 스캔
    │
    ▼
[신뢰도 임계값 체크]
    ├── 높음 → 응답 전송
    └── 낮음 → "확인 후 안내" + 에스컬레이션
```

## 핵심 결론

**금융 도메인 에이전트의 최대 위협은 기술 장애가 아니라 규제 위반과 잘못된 정보 제공입니다.**

현재 상태로 다음 주 배포는 **No-Go**입니다. RPN 150+ 실패 모드 7건이 모두 미대응 상태이며, 특히 **규정 변경 미반영(315)** 과 **잘못된 금융정보(300)** 는 단일 사건으로도 서비스 전체를 중단시킬 수 있습니다.

**권고**: No-Go 조건 7개 완료 후 소규모 파일럿(100건/일, 2주)으로 시작 → 이상 없을 시 전체 배포.
