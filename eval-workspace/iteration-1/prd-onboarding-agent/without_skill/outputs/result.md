# PRD: 신규 입사자 온보딩 가이드 에이전트

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 제품명 | 온보딩 가이드 에이전트 (OGA: Onboarding Guide Agent) |
| 작성일 | 2026-03-06 |
| 상태 | 초안 |
| 우선순위 | P0 |

### 1.1 배경 및 목적

신규 입사자는 첫 주에 수십 가지 정보를 동시에 습득해야 하며, 담당자에게 반복적인 질문이 집중되는 문제가 발생한다. OGA는 이 과정을 자동화하여 신규 입사자의 온보딩 경험을 개선하고 HR/IT 담당자의 반복 업무를 줄이는 것을 목표로 한다.

### 1.2 성공 지표 (KPI)

- 온보딩 관련 HR/IT 문의 건수 **30% 감소** (3개월 내)
- 신규 입사자 온보딩 만족도 점수 **4.0/5.0 이상**
- FAQ 자동 응답률 **70% 이상**
- 문서 검색 정확도 (상위 3개 결과 내 정답 포함) **80% 이상**

## 3. 핵심 기능 요구사항

### 3.1 F-01: 온보딩 체크리스트 안내

**설명:** 입사일 기준으로 D+1, D+3, D+7, D+14, D+30 타임라인에 맞춰 해야 할 일을 자동으로 Slack DM으로 발송한다.

**상세 요구사항:**
- 입사일 정보를 HR 시스템(또는 Slack 프로필)에서 자동으로 읽어온다
- 체크리스트 항목 완료 시 `/done [항목명]` 명령어로 상태 업데이트 가능
- 미완료 항목에 대해 D+N 당일 리마인더 발송
- 체크리스트 항목은 팀/직군별로 커스터마이징 가능

**예시 체크리스트 항목:**

```
[ D+1 ] 사원증 수령 및 출입카드 등록
[ D+1 ] 노트북 초기 설정 (IT 지원 요청)
[ D+3 ] 사내 시스템 계정 개설 확인 (Jira, Confluence, GitHub 등)
[ D+7 ] 팀 OKR 문서 열람
[ D+14 ] 30-60-90일 플랜 작성
```

---

### 3.2 F-02: FAQ 자동 응답

**설명:** 신규 입사자의 일반 질문에 대해 LLM 기반으로 즉시 답변한다.

**상세 요구사항:**
- Slack에서 에이전트를 멘션(`@onboarding-guide`)하거나 DM으로 질문
- FAQ 지식베이스(HR 정책, IT 안내, 복지 규정 등)를 RAG(Retrieval-Augmented Generation)로 검색 후 응답
- 답변 신뢰도가 낮을 경우 (`confidence < 0.7`) 담당자 연결 안내
- 답변 하단에 참조 문서 링크 포함
- 사용자가 답변에 👍/👎 이모지로 피드백 제공 시 로그 수집

**FAQ 카테고리 예시:**

| 카테고리 | 예시 질문 |
|----------|-----------|
| 급여/복지 | "연차는 몇 일인가요?", "식대 지원은 어떻게 신청하나요?" |
| IT/장비 | "VPN 설치는 어떻게 하나요?", "모니터 추가 신청 방법?" |
| 문화/규정 | "재택근무 정책이 어떻게 되나요?", "경비 처리 절차는?" |
| 조직 | "인사팀 담당자가 누구인가요?", "조직도는 어디서 보나요?" |

---

### 3.3 F-03: 문서 검색

**설명:** 키워드 또는 자연어 질의로 사내 문서를 검색하고 관련 문서 목록을 반환한다.

**상세 요구사항:**
- Slack 명령어: `/find [검색어]`
- 연동 대상 문서 소스: Confluence, Google Drive, Notion (우선순위 순)
- 검색 결과는 관련도 순으로 최대 5개 반환
- 각 결과에 문서 제목, 링크, 마지막 수정일, 담당자 표시
- 검색 결과 없을 경우 유사 키워드 제안

---

### 3.4 F-04: Slack 연동 (필수)

**상세 요구사항:**

| 기능 | 구현 방식 |
|------|-----------|
| DM 수신/발신 | Slack Events API (`message.im`) |
| 채널 멘션 응답 | Slack Events API (`app_mention`) |
| Slash 명령어 | `/onboarding`, `/find`, `/done` |
| 예약 메시지 발송 | Slack `chat.scheduleMessage` API |
| 이모지 반응 감지 | `reaction_added` 이벤트 |
| 홈 탭 | 체크리스트 현황 및 진행률 표시 |

**Slack App 권한 (Bot Token Scopes):**

```
channels:read, chat:write, commands, im:history,
im:read, im:write, reactions:read, users:read,
users:read.email, files:read
```

---

### 3.5 F-05: 관리자 대시보드

**설명:** HR/IT 운영자가 FAQ 지식베이스를 관리하고 온보딩 현황을 모니터링한다.

**상세 요구사항:**
- 웹 UI 기반 관리 콘솔
- FAQ 문서 업로드, 수정, 삭제 (파일 형식: PDF, DOCX, Markdown)
- 체크리스트 항목 및 타이밍 설정
- 팀/직군별 온보딩 템플릿 관리
- 주간 리포트: 질문 빈도, 미응답 질문, 만족도 통계
